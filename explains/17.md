## CRUD とは

**CRUD** は、データベース操作の4つの基本を表す略語です。

| 操作 | 英語 | SQL 文 | 説明 |
| ---- | ------ | --------- | ---------------------- |
| 作成 | Create | `INSERT` | データの新規作成 |
| 読み取り | Read | `SELECT` | データの取得 |
| 更新 | Update | `UPDATE` | データの更新 |
| 削除 | Delete | `DELETE` | データの削除 |

Web アプリケーションのほぼすべての機能はこの4つで構成されています。

#### 実行コード
```php
// ① SQL をプレースホルダ（:name）付きで用意する
$sql = "SELECT * FROM users WHERE id = :id";

// ② SQL を DB に「事前準備」させる
$stmt = $pdo->prepare($sql);

// ③ プレースホルダに値を渡して実行する
$stmt->execute(['id' => $id]);

// ④ 結果を取得する
$user = $stmt->fetch(PDO::FETCH_ASSOC);
```

### fetch と fetchAll の違い

| メソッド | 取得件数 | 用途 |
| ------------ | ------ | -------------------- |
| `fetch()` | 1件 | 詳細表示・ログイン |
| `fetchAll()` | 全件 | 一覧表示 |


### FETCH_ASSOC
どちらも `PDO::FETCH_ASSOC` を渡すと、カラム名をキーにした**連想配列**で返ります。

```php
stdClass Object
(
    [id] => 1
    [account_name] => "john_doe"
    [email] => "john@example.com"
    [display_name] => "John Doe"
)
```

### FETCH_OBJ
`FETCH_OBJ` を渡すと、オブジェクトで返ります。

```php
// FETCH_OBJ で取得した結果のイメージ
[
    'id'           => 1,
    'account_name' => 'john_doe',
    'email'        => 'john@example.com',
    'display_name' => 'John Doe',
]
```

## READ（一覧取得）― select_users.php

`SELECT` 文でテーブルの全レコードを取得します。

```php
function get($limit = 50)
{
    $pdo = Database::getInstance();

    $sql = "SELECT * FROM users LIMIT :limit;";
    $stmt = $pdo->prepare($sql);

    // LIMIT 句の値は整数型を明示して渡す
    $stmt->bindValue(':limit', (int)$limit, PDO::PARAM_INT);
    $stmt->execute();

    return $stmt->fetchAll(PDO::FETCH_ASSOC); // 全件を連想配列で返す
}
```

### LIMIT 句と bindValue

`LIMIT` や `OFFSET` に整数値を渡すときは `execute(['key' => val])` ではなく
`bindValue()` で**型を明示**する必要があります。

```php
$stmt->bindValue(':limit', (int)$limit, PDO::PARAM_INT);
// PDO::PARAM_INT 整数型を指定（省略すると文字列型になりエラーになることがある）
```

### SELECT 文の基本構文

```sql
SELECT カラム名          -- * はすべてのカラム
FROM テーブル名
WHERE 条件               -- 省略すると全件取得
ORDER BY カラム名 DESC   -- 並び順（DESC: 降順 / ASC: 昇順）
LIMIT 件数;              -- 最大取得件数
```

## READ（1件取得）― find_user.php

ID を指定して1件だけ取得するときは `fetch()` を使います。

```php
function find($id)
{
    $pdo = Database::getInstance();

    $sql = "SELECT * FROM users WHERE id = :id;";
    $stmt = $pdo->prepare($sql);
    $stmt->execute(['id' => $id]); // プレースホルダに値を渡す
    return $stmt->fetch(PDO::FETCH_ASSOC); // 1件だけ取得
}
```

## SQL インジェクションとは

**SQL インジェクション**は、ユーザーが入力した悪意ある文字列を SQL に直接埋め込むことで
意図しない SQL が実行されてしまう攻撃です。

### 危険なコード（対策なし）

```php
function findInjection($id)
{
    $pdo = Database::getInstance();
    // ❌ 入力値をそのまま SQL に埋め込んでいる
    $sql = "SELECT * FROM users WHERE id = $id;";
    $stmt = $pdo->query($sql);
    return $stmt->fetch(PDO::FETCH_ASSOC);
}
```

この `$id` に次の文字列が渡されたとします。

```
'' OR 1=1;--
```

実行される SQL はこうなります。

```sql
SELECT * FROM users WHERE id = '' OR 1=1;--
```

`OR 1=1` は常に**真（true）**なので、`WHERE` 条件が常に成立し、
**全ユーザーのデータが漏洩**します。

### 攻撃の仕組み



### SQL インジェクション対策
プリペアドステートメントを使うと、入力値は**文字列としてエスケープ**され
SQL の命令として解釈されなくなります。

> **プリペアドステートメントを使うことが SQL インジェクション対策の基本**

## CREATE（登録）― insert_user.php

`INSERT` 文でレコードを追加します。

```php
function insert($posts)
{
    // ① パスワードをハッシュ化してから保存する
    $posts['password'] = password_hash($posts['password'], PASSWORD_DEFAULT);

    $pdo = Database::getInstance();

    $sql = "INSERT INTO users (account_name, email, display_name, password)
            VALUES (:account_name, :email, :display_name, :password)";

    $stmt = $pdo->prepare($sql);
    $result = $stmt->execute($posts); // 配列のキーとプレースホルダが対応する

    if ($result) {
        return $pdo->lastInsertId(); // ② 登録直後の ID を取得して返す
    }
    return "登録処理に失敗しました。";
}
```

### password_hash()

パスワードはそのままDBに保存せず、必ず**ハッシュ化**します。
ハッシュ化とは元の文字列に戻せない一方向の変換です。

```php
$hash = password_hash('secret123', PASSWORD_DEFAULT);
// → "$2y$12$..." のような文字列になる（毎回異なる）
```

DBに万が一不正アクセスされても、元のパスワードはわかりません。

### lastInsertId()

```php
$pdo->lastInsertId(); // INSERT 直後に発行された AUTO_INCREMENT の ID を返す
```

### 一意制約違反のエラーハンドリング

`account_name` や `email` に `UNIQUE` 制約があるため、重複登録時はエラーになります。
`PDOException` のエラーコード `23000` で検知できます。

```php
} catch (PDOException $e) {
    if ($e->getCode() == 23000) {
        return "このアカウント名またはメールアドレスは既に登録されています。";
    }
    return "データベースエラー: " . $e->getMessage();
}
```

## UPDATE（更新）― update_user.php

`UPDATE` 文で既存レコードを上書きします。

```php
function update($id, $display_name, $password)
{
    $pdo = Database::getInstance();

    $params = ['id' => $id, 'display_name' => $display_name];

    // パスワードが入力されたときのみ更新対象に含める
    if (!empty($password)) {
        $hash = password_hash($password, PASSWORD_DEFAULT);
        $sql = "UPDATE users SET display_name = :display_name, password = :password WHERE id = :id;";
        $params['password'] = $hash;
    } else {
        $sql = "UPDATE users SET display_name = :display_name WHERE id = :id;";
    }

    $stmt = $pdo->prepare($sql);
    return $stmt->execute($params);
}
```

### UPDATE 文の基本構文

```sql
UPDATE テーブル名
SET カラム名 = 値, カラム名 = 値
WHERE 条件;   -- ← 必須！省略すると全件更新されてしまう
```

> `WHERE` 句を忘れると**テーブル全件が更新**されるため、必ず条件を指定する

## DELETE（削除）― delete_user.php

`DELETE` 文で指定したレコードを削除します。

```php
function delete($id)
{
    $pdo = Database::getInstance();
    $sql = "DELETE FROM users WHERE id = :id;";
    $stmt = $pdo->prepare($sql);
    return $stmt->execute(['id' => $id]);
}
```

### DELETE は POST メソッドでのみ受け付ける

削除は取り消せない操作なので、URL にアクセスするだけで実行されないよう
`POST` リクエストのみ受け付ける設計にしています。

```php
// POST でなければ、またはIDがなければ一覧へ戻す
if ($_SERVER['REQUEST_METHOD'] !== 'POST' || empty($_POST['id'])) {
    header('Location: select_users.php');
    exit;
}
```

### DELETE 文の基本構文

```sql
DELETE FROM テーブル名
WHERE 条件;   -- ← 必須！省略すると全件削除されてしまう
```

> `WHERE` 句を忘れると**テーブルの全データが消える**ため、最も注意が必要な操作

## JOIN（テーブル結合）― user_tweets.php

複数のテーブルを**結合**して、関連データをまとめて取得します。

`users` テーブルと `tweets` テーブルを `user_id` で紐付けています。

```php
function getByUserID($user_id, $limit = 20)
{
    $pdo = Database::getInstance();

    $sql = "SELECT
                tweets.id,
                users.display_name,
                users.account_name,
                tweets.message,
                tweets.created_at
            FROM tweets
            JOIN users ON tweets.user_id = users.id   -- ← JOIN でテーブルを結合
            WHERE tweets.user_id = :user_id
            ORDER BY tweets.created_at DESC            -- 新しい順に並べる
            LIMIT :limit";

    $stmt = $pdo->prepare($sql);
    $stmt->bindValue(':user_id', $user_id, PDO::PARAM_INT);
    $stmt->bindValue(':limit',   (int)$limit, PDO::PARAM_INT);
    $stmt->execute();

    return $stmt->fetchAll(PDO::FETCH_ASSOC);
}
```

「`tweets.user_id` と `users.id` が一致する行を結合する」という意味です。

## CRUD 操作の対応まとめ

| 操作 | SQL | PHP メソッド | 返り値 |
| ------ | --------- | ----------------------- | -------------------- |
| 全件取得 | `SELECT` | `prepare` → `execute` → `fetchAll` | 連想配列の配列 |
| 1件取得 | `SELECT` | `prepare` → `execute` → `fetch` | 連想配列（1件） |
| 登録 | `INSERT` | `prepare` → `execute` | 成否 bool |
| 更新 | `UPDATE` | `prepare` → `execute` | 成否 bool |
| 削除 | `DELETE` | `prepare` → `execute` | 成否 bool |

## まとめ

| 概念 | キーワード | 意味 |
| -------------------- | ---------------------------------------- | ------------------------------------ |
| CRUD | Create / Read / Update / Delete | DB操作の4基本 |
| プリペアドステートメント | `prepare()` / `execute()` | SQL を安全・効率的に実行する仕組み |
| SQL インジェクション | `' OR 1=1;--` | 悪意ある入力で SQL を改ざんする攻撃 |
| プレースホルダ | `:name` | SQL 中の値を埋め込む場所 |
| パスワードハッシュ | `password_hash()` | 元に戻せない一方向変換でパスワードを保存 |
| 最終挿入ID | `lastInsertId()` | INSERT 直後の AUTO_INCREMENT 値を取得 |
| テーブル結合 | `JOIN ... ON` | 複数テーブルを条件で結合して取得する |
