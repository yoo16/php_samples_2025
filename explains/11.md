## はじめに

特定の処理をひとまとめにして名前をつけたものを**関数（Function）**と呼びます。PHPが最初から用意している**ビルトイン関数**と、自分で作る**ユーザー定義関数**の両方を、注文処理（`order.php`）とデータチェック（`data_check.php`）の実例で学びましょう。

## PHPビルトイン関数（標準関数）

PHPには、データのチェックや文字列の加工など、便利な道具が最初からたくさん用意されています。

### データ検証関数

| 関数名 | 説明 | true になる条件 | 注意点 |
| - | - | - | - |
| `isset()` | 変数が存在し、かつ `null` ではないかを確認 | 変数が定義済み & `null` 以外 | 未定義変数対応 |
| `empty()` | 空とみなされる値かを確認 | `""`, `"0"`, `0`, `0.0`, `null`, `false`, `[]` など | `"0"` も true |
| `is_string()` | 文字列型か確認 | string 型のみ | 数値文字列 `"123"` は true |
| `is_numeric()` | 数値または数値文字列か確認 | `10`, `3.14`, `"10"`, `"3.14"` | `"10abc"` は false |
| `is_null()` | `null` か確認 | 値が `null` | 未定義変数は Warning になる |
| `is_int()` | 整数型か確認 | int 型のみ | `"10"` は false |
| `is_float()` | 浮動小数点型か確認 | float 型のみ | `"3.14"` は false |
| `is_bool()` | 真偽値型か確認 | true / false | 0 や 1 は false |
| `is_array()` | 配列型か確認 | array 型のみ | オブジェクトは false |
| `is_object()` | オブジェクト型か確認 | object 型のみ | null は false |

#### 比較方法

| 比較方法 | 型変換 | 例 | 結果 |
| - | - | - | - |
| `==` | あり（型ジャグリング） | `"10" == 10` | true |
| `===` | なし（型も比較） | `"10" === 10` | false |

#### 検証結果

| 値 | empty() | isset() | is_null() |
| - | - | - | - |
| `""` | true | true | false |
| `"0"` | true | true | false |
| `0` | true | true | false |
| `null` | true | false | true |
| `false` | true | true | false |
| `[]` | true | true | false |
| `"hello"` | false | true | false |
| 未定義変数 | true | false | ⚠ Warning |

### データ検証

`data_check.php` では、テスト用の文字列に対してこれらの関数を適用し、結果を `$results` 配列にまとめて表示しています。

```php
<?php
$testValue = $_GET['v'] ?? '  Hello, PHP Tokyo!  ';

// 1. 変数・データチェック関数
$results['check'] = [
    'isset'      => isset($testValue),       // 変数が存在し null でないか
    'empty'      => empty($testValue),       // 空とみなされるか（"" / 0 / null / [] など）
    'is_string'  => is_string($testValue),   // 文字列型か
    'is_numeric' => is_numeric($testValue),  // 数値または数値文字列か
    'is_null'    => is_null($testValue),     // null か
];
```

### 文字列操作関数

```php
<?php
$rawString     = $testValue;
$trimmedString = trim($rawString); // 前後の空白を除去

// 2. 文字列の長さ
$results['string'] = [
    'strlen'    => strlen($rawString),         // バイト数（日本語は1文字3バイト）
    'mb_strlen' => mb_strlen($rawString),      // 文字数（マルチバイト対応）
    'trimmed'   => $trimmedString,
    'upper'     => strtoupper($trimmedString), // 大文字に変換
    'lower'     => strtolower($trimmedString), // 小文字に変換
];

// 3. 部分文字列の抽出
$results['substr'] = [
    'substr_5'    => substr($trimmedString, 0, 5),    // 先頭5バイト
    'mb_substr_2' => mb_substr($trimmedString, 0, 2), // 先頭2文字（マルチバイト対応）
];

// 4. 置換・検索
$results['replace'] = [
    'replace' => str_replace('PHP', '🐘', $trimmedString), // 文字列を置換
    'pos'     => mb_strpos($trimmedString, 'PHP'),          // 位置を検索（なければ false）
];
```

### 数値操作関数

```php
<?php
// 5. 数値操作（入力が数値のときのみ意味を持つ）
$number = $results['check']['is_numeric'] ? $testValue : 0;

$results['number'] = [
    'format' => number_format($number), // 3桁カンマ区切り
    'ceil'   => ceil($number),          // 切り上げ
    'floor'  => floor($number),         // 切り捨て
    'round'  => round($number),         // 四捨五入
];
```

> **マルチバイト関数について**
> 日本語などの全角文字を扱う場合は、先頭に `mb_` がつく関数（マルチバイト関数）を使いましょう。`strlen()` はバイト数、`mb_strlen()` は文字数を返します。「東京」は `strlen` で 6、`mb_strlen` で 2 になります。

## ユーザー定義関数（自分で作る関数）

`function` キーワードを使って独自の関数を定義できます。`order.php` では注文処理に必要な計算をそれぞれ関数として切り出しています。モダンな PHP では引数と戻り値に**型**を指定することで、より安全なコードが書けます。

```php
<?php
// 定数（スコープをまたいでどこからでも参照できる）
const TAX_RATE   = 0.1;
const POINT_RATE = 0.05;

/**
 * ユーザーをIDで検索
 * ? をつけると「null を返す可能性がある」ことを示す（Null許容型）
 */
function findUser(int $id): ?array
{
    require_once "data/users.php";
    $ids   = array_column($users, "id");
    $index = array_search($id, $ids);
    return ($index !== false) ? $users[$index] : null;
}

/**
 * 商品をIDで検索
 */
function findProduct(int $id): ?array
{
    require_once "data/products.php";
    $ids   = array_column($products, "id");
    $index = array_search($id, $ids);
    return ($index !== false) ? $products[$index] : null;
}

/**
 * 合計金額の計算（単価 × 数量）
 * 引数に int、戻り値にも : int を指定して型を厳格にする
 */
function calculateSubtotal(int $price, int $quantity): int
{
    return $price * $quantity;
}

/**
 * ポイントの計算
 * デフォルト引数：$rate を省略すると POINT_RATE（0.05）が使われる
 */
function calculatePoint(int $amount, float $rate = POINT_RATE): int
{
    return (int)floor($amount * $rate);
}

// 呼び出し
$user    = findUser(1);        // ['id'=>1, 'display_name'=>'Chris Johnson', ...]
$product = findProduct(1);     // ['id'=>1, 'name'=>'クラフトコーラ', 'price'=>750, ...]

if ($user && $product) {
    $subtotal     = calculateSubtotal($product['price'], 3); // 750 × 3 = 2250
    $earnedPoints = calculatePoint($subtotal);               // floor(2250 × 0.05) = 112
}
```

| 要素 | 意味 | 例 |
| :--- | :--- | :--- |
| **引数** | 関数に渡す入力値。型を指定できる | `int $price` |
| **デフォルト引数** | 省略したときに使われる値 | `float $rate = POINT_RATE` |
| **戻り値の型** | 関数が返す値の型 | `: int` |
| **Null許容型** | その型または null を許容する | `: ?array` |

> **DocBlock コメント**
> 関数の直前に `/** ... */` 形式で書くコメントを **DocBlock** といいます。引数・戻り値の説明を書いておくと、IDE が補完に活用してくれます。チームでの開発では書くことを習慣にしましょう。

## 無名関数とアロー関数

名前をつけずに変数に代入する**無名関数（クロージャ）**と、1行で書けるシンプルな**アロー関数**（PHP 7.4+）も、モダンな PHP では頻繁に使われます。`order.php` ではメッセージ生成と税抜き計算にそれぞれ使っています。

```php
<?php
/**
 * メッセージの作成（無名関数 / クロージャ）
 * 変数に代入して使う。外部変数が必要なら use で取り込む。
 */
$formatGreeting = function (string $name): string {
    return "{$name}さん、この商品を購入しますか？";
};

// $user['display_name'] を渡して呼び出し
echo $formatGreeting($user['display_name']); // "Chris Johnsonさん、この商品を購入しますか？"

/**
 * 税抜き価格の計算（アロー関数）
 * fn キーワードで1行に書ける。外部の定数 TAX_RATE に use なしでアクセス可。
 */
$getExclTaxPrice = fn(int $inclTaxPrice): int => (int)ceil($inclTaxPrice / (1 + TAX_RATE));

echo $getExclTaxPrice($product['price']); // ceil(750 / 1.1) = 682（税抜き価格）

// array_filter() や array_map() との組み合わせが典型的な使い方
$expensive = array_filter($products, fn($p) => $p['price'] >= 700);
$names     = array_map(fn($p) => $p['name'], $products);
```

- **無名関数** `function() {}` — 変数に入れて使う・他の関数に渡す（コールバック）。外部変数を使いたい場合は `use` が必要。
- **アロー関数** `fn() =>` — 1行の戻り値のみの処理に使う。外部変数に `use` なしでアクセスできる。PHP 7.4 以降。

## 変数のスコープ（有効範囲）

PHPでは、関数の**外で定義した変数は関数の中では使えません**。これを**スコープ**といいます。`order.php` の `findUser()` や `findProduct()` が関数内で `require_once` を呼び出しているのもこのためです。

```php
<?php
const TAX_RATE = 0.1; // 定数はどこからでもアクセス可能

$taxRate = 0.1; // グローバルスコープの変数

function calcTax(int $price): float
{
    // $taxRate はここでは使えない（関数スコープ）
    // → 引数で受け取るか、定数を使う
    return $price * TAX_RATE; // 定数は $ なしで直接参照できる
}

// 外部変数を関数内で使いたい場合：use で取り込む（無名関数のみ）
$calc = function(int $price) use ($taxRate): float {
    return $price * $taxRate; // use で取り込んだ変数は使える
};

echo calcTax(750);  // 75.0
echo $calc(750);    // 75.0
```

> **定数はスコープを超えられる**
> `const TAX_RATE = 0.1` のように定義した定数は、関数の中からでも `$` なしで直接参照できます。`order.php` の `TAX_RATE` と `POINT_RATE` がまさにこのパターンです。

## まとめ

- **ビルトイン関数** — `isset()` / `empty()` でデータ検証、`mb_strlen()` / `str_replace()` で文字列操作、`number_format()` / `ceil()` で数値処理。
- **ユーザー定義関数** — `function` キーワードで定義。引数と戻り値に型を指定すると安全。`?型` で null を許容できる。
- **デフォルト引数** — 省略可能な引数を設定できる。`POINT_RATE` のように定数をデフォルト値にすると便利。
- **無名関数** — 変数に代入して使う。`use` で外部変数を取り込む。
- **アロー関数** `fn()` — 1行の処理をシンプルに書ける。外部スコープの変数・定数に `use` なしでアクセスできる。PHP 7.4 以降。
- 関数の中から外の変数は見えない（**スコープ**）。定数はどこからでもアクセス可能。
