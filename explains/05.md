## はじめに

情報を一時的に保存する「変数」、データの種類を表す「型」、そしてPHPが最初から用意している「スーパーグローバル変数」を学びましょう。

## 変数の宣言と代入

PHPでは `$` 記号を変数名の先頭につけます。`=` は「等しい」ではなく「右の値を左の変数に入れる（代入する）」という命令です。

```php
<?php
// 変数の代入（= は「代入演算子」）
$name   = "ユーザーA";   // string（文字列）
$level  = 5;             // int（整数）
$active = true;          // bool（論理型）

// 変数の中身を画面に出力する
echo $name;   // → ユーザーA

// 短縮形（HTMLの中でよく使う）
// <?= $name ?> は <?php echo $name; ?> と同じ
```

> **命名ルール：**
> 変数名は `$` の直後から始まり、数字からは始められません。大文字・小文字は区別されます（`$Name` と `$name` は別物）。

## データ型（Data Types）

変数に入れるデータの「種類」のことをデータ型と呼びます。PHPには次の基本型があります。

| 型名 | 種類 | 値の例 |
| :--- | :--- | :--- |
| **`string`** | 文字列 | `"ユーザーA"`, `'PHP'` |
| **`int`** | 整数 | `5`, `-10`, `0` |
| **`float`** | 浮動小数点数 | `1.5`, `3.14` |
| **`bool`** | 論理型（真偽値） | `true`, `false` |
| **`null`** | 値なし | `null` |
| **`array`** | 配列・連想配列 | `["剣", "盾"]`, `["hp" => 100]` |

```php
<?php
// 基本型
$name      = "ユーザーA";          // string
$level     = 5;                    // int
$exp_rate  = 1.5;                  // float
$is_active = true;                 // bool
$last_login = null;                // null

// 配列（インデックス配列）
$items = ["剣", "盾", "回復薬"];
echo $items[0]; // → 剣

// 連想配列（キーと値のペア）
$status = [
    "hp"  => 100,
    "mp"  => 50,
    "job" => "戦士",
];
echo $status["job"]; // → 戦士
```

## `var_dump()` でデバッグ

`var_dump()` は変数の **型** と **値** を同時に表示してくれるデバッグの定番関数です。「なんかおかしい」と思ったらまずこれで確認しましょう。

```php
<?php
$v1 = "100";   // string
$v2 = 100;     // int
$v3 = 100.0;   // float

var_dump($v1); // → string(3) "100"
var_dump($v2); // → int(100)
var_dump($v3); // → float(100)
```

> **使い分け：** 値を確認したいだけなら `echo`、型まで含めて調べたいなら `var_dump()` が便利です。

## 型の比較と型ジャグリング

PHPの比較演算子には **値だけ** を比べる `==` と、**型まで含めて** 比べる `===` の2種類があります。この違いはバグの原因になりやすいので注意が必要です。

```php
<?php
$v1 = "100";  // string
$v2 = 100;    // int

var_dump($v1 == $v2);  // → bool(true)  ← 値だけ比べると同じ
var_dump($v1 === $v2); // → bool(false) ← 型が違うので不一致
```

### 型ジャグリング（Type Juggling）

また、PHPは計算などで型が異なる場合に自動で変換する **型ジャグリング（Type Juggling）** という仕組みを持っています。

```php
<?php
$v1 = "100"; // string

// string と int を足すと、string が自動的に int として扱われる
var_dump($v1 + 50); // → int(150)
```

> **注意：** 意図しない型変換はバグの元になります。型が重要な比較では、常に `===`（厳密な比較）を使う習慣をつけましょう。

### 自動キャスト

プログラムの仕様によっては自動キャストとも言いますが、PHPとは若干異なります。

|        | 自動キャスト  | 型ジャグリング       |
| ------ | ------- | ------------- |
| 意味     | 暗黙的な型変換 | 型変換が複雑に連鎖する挙動 |
| レベル    | 言語機能    | その結果起きる現象     |
| ニュアンス  | 中立      | ややネガティブ       |
| 主に使う言語 | 多くの言語   | 主にPHP         |

## 特別な変数「スーパーグローバル」

PHPがあらかじめ用意している特別な変数で、どこからでもアクセスできます。どれも **連想配列** として値を保持しています。

- **`$_GET`**: URLの末尾に付いたパラメータ（`?name=Taro` など）を取得します。
- **`$_POST`**: フォームから `method="post"` で送信されたデータを取得します。URLには表示されません。
- **`$_SERVER`**: サーバーやリクエストの情報（サーバー名・IPアドレス・リクエスト方法・User-Agent など）を取得します。

```php
<?php
// $_GET: URLパラメータの取得
// URL例: superglobals.php?name=Yamada
$name = $_GET['name'] ?? 'ゲスト'; // キーがなければ 'ゲスト'

// $_POST: フォームデータの取得
$message = $_POST['message'] ?? '';

// $_SERVER: サーバー情報の取得
$method     = $_SERVER['REQUEST_METHOD']; // "GET" or "POST"
$ip         = $_SERVER['REMOTE_ADDR'];    // アクセス元のIPアドレス
$user_agent = $_SERVER['HTTP_USER_AGENT']; // ブラウザ情報
```

> **`??` とは？**
> Null合体演算子（Null Coalescing Operator）といいます。左辺が `null` またはキーが存在しない場合に右辺の値を返します。スーパーグローバルと組み合わせてよく使います。

## 定数（Constants）

値を変えたくない場合は、変数ではなく **定数** を使います。定数には `$` は不要で、慣習として **大文字スネークケース** で名前をつけます。

```php
<?php
// const で定数を定義（クラス内でも使える）
const TAX_RATE    = 0.1;
const SITE_NAME   = "PHPサンプル";

// define() でも定義できる（実行時に決まる値に使う）
define('MAX_LOGIN_ATTEMPTS', 5);

// 定数は $ なしで呼び出す
echo TAX_RATE;  // → 0.1
echo SITE_NAME; // → PHPサンプル
```

> **変数との違い：** 定数は一度定義したら上書きできません。税率・サイト名・APIエンドポイントなど、アプリ全体で固定すべき値に使います。

## まとめ

- 変数は `$` で始まり、データを一時的に保存する。
- データ型には string, int, float, bool, array, null などがある。
- `var_dump()` は型 and 値を確認できるデバッグの基本。
- 型まで含めた比較には `===` を使う。
- スーパーグローバル変数は `$_GET` や `$_POST` など、どこからでも使える連想配列。
- 定数は `const` や `define()` で定義し、一度決めたら変更できない。
