# PHP SNS アプリで学ぶ Webアプリ開発

## はじめに

実際に動く SNS アプリのコードを読み、機能を追加しながら、本格的な Web アプリの作り方を学びます。
教科書で学んだ PHP・MySQL・オブジェクト指向の知識を、**実際のプロダクトのコードで使いこなす**ことがゴールです。

---

## 学ぶこと

### MVC アーキテクチャ
Web アプリの設計パターン「MVC」を実際のコードで理解します。
Controller・Model・View それぞれの役割と、なぜ分けて書くのかを体感します。

### 機能追加の実践
いいね機能・返信機能・ルーティングなど、既存のコードに新しい機能を追加する経験をします。
「読む → 理解する → 真似る → 自分で書く」の流れを繰り返します。

### 設計を自分で考える
機能を追加するとき、何をどう設計するかを自分で考えて
**仕様書**にまとめます。
設計力は、どんな言語・フレームワークでも使える一生もののスキルです。

### フレームワークへの準備
Laravel などのフレームワークを学ぶ前に、「フレームワークが何を自動化してくれているのか」を理解します。
このアプリで手書きしたコードが、フレームワークではどう変わるかを比較します。

---

## 学習の進め方

### 1：読む・理解する
アプリを動かして、コードの全体像を把握します。
MVC の3つの役割をホワイトボードや紙などに図で書いたり書類をまとめて、理解することが目標です。

### 2：小さく追加する
既存の機能のコードを読んで、パターンを理解します。
その後、返信機能を仕様書をもとに実装します。

### 3：設計する
クリーンURL（`user/5` のような形式）を実現するルーティングを実装します。
さらに、自分でオリジナル機能を1つ考えて仕様書を書きます。

### 4：作る・発表する
自分が設計した機能を実装して、クラスに発表します。
Laravel との比較を通じて、次のステップへの準備をします。

---

### 重要なルール

**仕様書は必ず自分で書く。**

AI はコードを書いてくれますが、「何を作るか・どう設計するか」は自分で考えなければなりません。
仕様書を書く力 ＝ 設計力を鍛えることが、このコースの核心です。

### プロジェクトのルールファイル

Vibe Coding はプロジェクト内の以下のファイルを読んで、
このアプリに合ったコードを書いてくれます。

| ファイル | 内容 |
|---|---|
| `CLAUDE.md` | プロジェクトの全体像・ディレクトリ構成 |
| `SKILL.md` | 実装パターンの辞書（Model・Controller・View の書き方） |
| `.claude/rules/` | セキュリティ・命名規則などのルール集 |
