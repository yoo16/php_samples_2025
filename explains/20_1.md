# SNSアプリ開発：設計書・仕様書

このドキュメントは、PHP（フレームワーク不使用）で構築されたSNSアプリの仕組みを理解するための教材用設計資料です。

---

## 第1章：はじめに（仕様書の読み方と用語解説）

### 仕様書・設計書とは？
「どんな機能を作るのか（仕様）」「それをどうやってプログラムで実現するのか（設計）」をまとめた、開発の地図となるドキュメントです。迷った時はこの地図に戻って全体像を確認しましょう。

### 覚えておきたい開発用語
| 用語 | 解説 |
| :--- | :--- |
| **Vanilla PHP** | Laravelなどの「フレームワーク」を使わず、素のPHPだけで書かれたプログラムのこと。基礎を学ぶのに最適です。 |
| **MVC** | **M**odel（データ処理）、**V**iew（画面表示）、**C**ontroller（全体の司令塔）の3つに役割を分ける設計手法です。 |
| **AJAX** | 画面全体を再読み込みせずに、裏側でサーバーと通信する技術。このアプリでは「いいね」ボタンなどで使われています。 |
| **エンドポイント** | ブラウザやJavaScriptがアクセスする「URLの終着点（窓口）」のことです。 |
| **ルーティング** | 「このURLにアクセスしたら、このファイルを実行する」という道案内の仕組みです。 |

## 第2章：プロジェクト概要と環境構築

### プロジェクト概要
PHP製のシンプルなSNSアプリケーションです。MVC設計パターンに従っており、Webアプリの基本となる認証・データベース連携・画像の扱い・AJAX通信を実践的に学ぶことができます。

### 実装されている機能一覧
- ユーザー登録 / ログイン / ログアウト
- プロフィール編集・プロフィール画像アップロード
- ツイート投稿・削除・画像付き投稿
- ツイート検索（キーワード・ハッシュタグ）
- いいね 追加・解除（AJAX）
- フォロー・アンフォロー
- フォロー数・フォロワー数表示
- ギャラリービュー（画像ツイート一覧）

### 技術スタック
| 項目 | 内容 |
| :--- | :--- |
| **Language** | PHP 8.x (Vanilla PHP, フレームワークなし) |
| **Database** | MySQL / MariaDB |
| **Frontend** | Vanilla JavaScript, Tailwind CSS (CDN), SVG |
| **Architecture** | MVC (Model-View-Controller) |

### セットアップ手順

1. **環境設定ファイルの作成**
   `env.sample.php` をコピーして `env.php` を作成し、自身の環境に合わせてデータベース接続情報を書き換えます。
   
   ```bash
   cp env.sample.php env.php
   ```

   **env.php の設定例:**

   | 定数名 | 説明 |
   | :--- | :--- |
   | `DB_CONNECTION` | 使用するデータベースの接続方法（mysql, pgsql, sqlite など） |
   | `DB_DATABASE` | 使用するデータベース名 |
   | `DB_USERNAME` | データベースのユーザー名 |
   | `DB_PASSWORD` | データベースのパスワード（XAMPPなら空、MAMPなら root 等） |
   | `DB_HOST` | データベースのホスト名（通常は 127.0.0.1, localhost） |
   | `DB_PORT` | データベースのポート番号（通常は 3306） |
   | `DB_CHARSET` | データベースの文字コード（UTF-8推奨） |
   | `APP_KEY` | アプリケーションキー（任意） |
   | `SITE_TITLE` | サイトタイトル（任意） |
   | `BASE_URL` | アプリを配置したURL（例: `http://localhost/20_php_sns/`） 設定しない場合は、自動検出|

2. **データベースの初期化**
   データベースを準備し、テーブルと初期データを入れます。

### mysqlコマンドの場合
```bash
# データベースとテーブルの作成
mysql -u [user] -p [db_name] < database/schema.sql

# テスト用データの挿入
mysql -u [user] -p [db_name] < database/insert_data.sql
```

### MVCの概念と処理フロー

ブラウザからURLにアクセスした時、プログラムは以下の順番でバケツリレーを行います。

| 層 (Layer) | 役割 (Role) | 具体的な処理の流れ (Flow) | 代表的なファイル |
| --- | --- | --- | --- |
| **Action** | **受付窓口** | アクセスを受け取り、Controllerを起動する。 | `home/index.php` |
| **Controller** | **司令塔** | 何をするか判断し、Modelにデータを依頼。結果をViewへ渡す。 | `HomeController.php` |
| **Model** | **データ担当** | データベース操作（SQL実行）やデータの加工を行う。 | `app/models/Tweet.php` |
| **View** | **表示担当** | 届いたデータをHTMLに埋め込み、ブラウザに画面を出力。 | `layout.view.php` |

### DDL（データ定義言語）

```sql
CREATE TABLE users (
    id bigint PRIMARY KEY AUTO_INCREMENT,
    account_name varchar(255) UNIQUE NOT NULL,
    email varchar(255) UNIQUE NOT NULL,
    display_name varchar(255) NOT NULL,
    password varchar(255) NOT NULL,
    profile text DEFAULT NULL,
    profile_image text DEFAULT NULL,
    created_at datetime NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

CREATE TABLE tweets (
    id bigint PRIMARY KEY AUTO_INCREMENT,
    message text NOT NULL,
    user_id bigint NOT NULL,
    image_path text DEFAULT NULL,
    created_at timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

CREATE TABLE likes (
    id bigint PRIMARY KEY AUTO_INCREMENT,
    user_id bigint NOT NULL,
    tweet_id bigint NOT NULL,
    created_at timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE (user_id, tweet_id)
);

-- 外部キー制約
ALTER TABLE tweets ADD CONSTRAINT tweets_user_id_fk
    FOREIGN KEY (user_id) REFERENCES users(id)
    ON DELETE CASCADE ON UPDATE CASCADE;

ALTER TABLE likes ADD CONSTRAINT likes_tweet_id_fk
    FOREIGN KEY (tweet_id) REFERENCES tweets(id)
    ON DELETE CASCADE ON UPDATE CASCADE;

ALTER TABLE likes ADD CONSTRAINT likes_user_id_fk
    FOREIGN KEY (user_id) REFERENCES users(id)
    ON DELETE CASCADE ON UPDATE CASCADE;

CREATE TABLE follows (
    id bigint PRIMARY KEY AUTO_INCREMENT,
    follower_id bigint NOT NULL,
    followee_id bigint NOT NULL,
    created_at timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    UNIQUE (follower_id, followee_id)
);

ALTER TABLE follows ADD CONSTRAINT follows_follower_id_fk
    FOREIGN KEY (follower_id) REFERENCES users(id)
    ON DELETE CASCADE ON UPDATE CASCADE;

ALTER TABLE follows ADD CONSTRAINT follows_followee_id_fk
    FOREIGN KEY (followee_id) REFERENCES users(id)
    ON DELETE CASCADE ON UPDATE CASCADE;

CREATE TABLE replies (
    id bigint PRIMARY KEY AUTO_INCREMENT,
    tweet_id bigint NOT NULL,
    user_id bigint NOT NULL,
    message text NOT NULL,
    created_at timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

ALTER TABLE replies ADD CONSTRAINT replies_tweet_id_fk
    FOREIGN KEY (tweet_id) REFERENCES tweets(id)
    ON DELETE CASCADE ON UPDATE CASCADE;

ALTER TABLE replies ADD CONSTRAINT replies_user_id_fk
    FOREIGN KEY (user_id) REFERENCES users(id)
    ON DELETE CASCADE ON UPDATE CASCADE;
```

## 第7章：開発のルールと共通設定
### ✅ 新機能追加時のチェックリスト

新しい画面や機能を作る時は、以下の手順でファイルを作成・更新します。

* [ ] `app/models/` に必要に応じてモデル（データ処理）を作成する。
* [ ] `app/controllers/` にコントローラー（司令塔）を作成・追記する。
* [ ] クラスを新設した場合、`app.php` に `require_once` を追記して読み込ませる。
* [ ] `[新機能名]/index.php` などのエントリポイント（受付窓口）を作成し、`app.php` を require する。
* [ ] 画面が必要なら `app/views/[機能名]/xxx.view.php` を作成する。
* [ ] AJAX APIが必要なら `api/[機能名]/[処理]/index.php` を作成する。
* [ ] JSファイルを追加した場合は、Viewのヘッダーなどで読み込む。

### 🛡 セキュリティの基本

Webアプリを攻撃から守るための絶対ルールです。

1. **安全な出力（XSS対策）**:
画面（View）に変数を出力する時は、必ず `h()` 関数（`Sanitize.php` で定義）で囲み、悪意のあるスクリプトを防ぎます。
2. **安全な入力（CSRF対策）**:
フォームなどで入力値を受け取る時は、サーバ側で CSRFトークンを作成し、フォームに埋め込み、送信時に検証します。
3. **安全なデータベース操作（SQLインジェクション対策）**:
SQLを実行する際は、変数（入力値）を直接SQL文に混ぜず、必ず `PDO` のプレースホルダー（プリペアドステートメント）を使用します。
