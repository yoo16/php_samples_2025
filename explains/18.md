## セッションとログイン認証

Web アプリケーションでは「誰がログインしているか」をページをまたいで記憶する仕組みが必要です。
この章では **セッション**（ログイン状態の管理）と **パスワード検証**（認証）を学びます。

## セッションとは

**セッション**は、ページをまたいでデータを保持する仕組みです。

HTTP はもともと「状態を持たない（ステートレス）」プロトコルです。
ページが変わるたびに「誰がリクエストしているか」がわからなくなります。
セッションを使うと、サーバー側にデータを保存し、**Cookie のセッションID** で紐付けることで状態を維持できます。

### session_start() と session_regenerate_id()

```php
// app.php
session_start();              // セッションを開始（または既存セッションを再開）
session_regenerate_id(true);  // セッション固定攻撃対策：セッションIDを再生成
```

| 関数 | 役割 |
| ---- | ---- |
| `session_start()` | セッションを開始。`$_SESSION` が使えるようになる |
| `session_regenerate_id(true)` | 古いセッションIDを無効化し、新IDを発行する |

> **セッション固定攻撃（Session Fixation）対策**：攻撃者が用意したIDをユーザに使わせ
> ログイン後もそのIDで不正アクセスする攻撃。ログイン時にIDを再生成して防ぐ。

## 共通ファイル（app.php）

`app.php` はすべてのページで `require_once` する**共通設定ファイル**です。

```php
// app.php
session_start();
session_regenerate_id(true);

const SITE_TITLE   = "Sign in";
const BASE_DIR     = __DIR__;
const LIB_DIR      = BASE_DIR . "/lib/";
const COMPONENT_DIR = BASE_DIR . "/app/components/";
const MODEL_DIR    = BASE_DIR . "/app/models/";
const SESSION_SECRET = "mysecretkey";

require_once BASE_DIR . '/env.php';       // DB 接続情報
require_once LIB_DIR  . 'Database.php';   // PDO 接続
require_once LIB_DIR  . 'Model.php';      // モデル基底クラス
require_once MODEL_DIR . 'User.php';      // User モデル
require_once MODEL_DIR . 'AuthUser.php';  // セッション管理クラス
```

`app.php` をまとめることで、各ページは `require_once "../app.php"` の1行だけで
必要なすべての設定・クラスが使えるようになります。

## Model 基底クラス（lib/Model.php）

`Model` クラスはすべてのモデルが継承する**基底クラス**です。

```php
class Model
{
    protected static function pdo()
    {
        return Database::getInstance(); // PDO インスタンスを返す
    }
}
```

継承先のモデルで `self::pdo()` と書くだけで PDO に接続でき、
接続コードを各モデルで繰り返す必要がなくなります。

```php
class User extends Model
{
    public function findByEmail($email)
    {
        $stmt = self::pdo()->prepare('SELECT ... FROM users WHERE email = ?');
        // ↑ Model::pdo() = Database::getInstance() が返す PDO を使う
    }
}
```

## User モデル（app/models/User.php）

`User` クラスはユーザのデータ取得と認証を担当します。

### findByEmail() — メールアドレスでユーザを検索

```php
public function findByEmail($email)
{
    $stmt = self::pdo()->prepare(
        'SELECT id, display_name AS name, email, profile_image AS image, password
         FROM users WHERE email = ? LIMIT 1'
    );
    $stmt->execute([$email]);
    return $stmt->fetch(PDO::FETCH_ASSOC) ?: null;
}
```

プレースホルダに `?`（位置パラメータ）を使うと、`execute()` に**配列の値**を順番に渡せます。

| プレースホルダの種類 | 書き方 | execute() の引数 |
| ------- | ---- | --------------- |
| 名前付き | `:email` | `['email' => $val]` |
| 位置 | `?` | `[$val]` |

### auth() — パスワード検証と認証

```php
public function auth($email, $password)
{
    $user = $this->findByEmail($email);        // ① メールアドレスでユーザを取得
    if ($user && password_verify($password, $user['password'])) {
        unset($user['password']);              // ② パスワードハッシュを取り除く
        return $user;                          // ③ 認証成功：ユーザデータを返す
    }
    return null;                               // ④ 認証失敗
}
```

### password_verify()

`password_verify()` は、平文パスワードとハッシュを比較する関数です。

```php
// 登録時: password_hash() でハッシュを生成して保存
$hash = password_hash('secret123', PASSWORD_DEFAULT);  // → "$2y$12$..."

// ログイン時: password_verify() でハッシュと比較
$ok = password_verify('secret123', $hash);  // → true
$ng = password_verify('wrong',     $hash);  // → false
```

> ハッシュは毎回異なる文字列になりますが、`password_verify()` は正しく比較できます。
> DBから取得したハッシュと文字列比較（`===`）しても絶対に一致しないので必ずこの関数を使います。

## AuthUser クラス（app/models/AuthUser.php）

`AuthUser` はセッションへのユーザ情報の**保存・取得・削除**を一元管理するクラスです。

```php
class AuthUser extends User
{
    private static $key = SESSION_SECRET; // セッションキー（"mysecretkey"）

    // セッションからユーザ情報を取得
    public static function check()
    {
        if (isset($_SESSION[self::$key])) {
            return $_SESSION[self::$key];
        }
    }

    // セッションにユーザ情報を保存
    public static function set($user)
    {
        $_SESSION[self::$key] = $user;
    }

    // セッションからユーザ情報を削除
    public static function clear()
    {
        if (isset($_SESSION[self::$key])) {
            unset($_SESSION[self::$key]);
        }
    }
}
```

| メソッド | 役割 | 呼び出し元 |
| -------- | ---- | -------- |
| `AuthUser::set($user)` | ログイン成功時にセッション保存 | `auth.php` |
| `AuthUser::check()` | ログイン中かどうかを確認 | `home/index.php` |
| `AuthUser::clear()` | ログアウト時にセッション削除 | `signout/index.php` |

## 認証処理（signin/auth.php）

`auth.php` は POST でフォームデータを受け取り、認証を実行します。

```php
require_once '../app.php';

if ($_SERVER['REQUEST_METHOD'] !== 'POST') {
    exit; // POST 以外は何もしない
}

// セッションにフォーム入力値を一時保存（再表示用）
$_SESSION['signin'] = $_POST;

$email    = $_POST['email'];
$password = $_POST['password'];

// ① User モデルで認証
$user = new User();
$auth_user = $user->auth($email, $password);

if (empty($auth_user['id'])) {
    // ② 失敗: エラーメッセージをセッションに保存してフォームへ戻す
    $_SESSION['error'] = 'メールアドレスまたはパスワードが間違っています。';
    header('Location: input.php');
    exit;
} else {
    // ③ 成功: セッションにユーザ情報を保存してホームへ
    AuthUser::set($auth_user);
    header('Location: ../home/');
    exit;
}
```

## ログインフォームの入力保持

入力エラー時、フォームに前回の値を**再表示**できます。

```php
// input.php
$form  = $_SESSION['signin'] ?? null;  // 直前の POST データ
$error = $_SESSION['error']  ?? null;  // エラーメッセージ
unset($_SESSION['error']);             // 1度表示したら削除（フラッシュメッセージ）
```

```html
<input type="text" name="email" value="<?= $form['email'] ?? '' ?>">
```

### フラッシュメッセージ

**フラッシュメッセージ**とは、「1度だけ表示して消えるメッセージ」のことです。
セッションに保存 → 読み取り後に `unset()` で削除するパターンで実装します。

```php
// auth.php: エラーをセッションに保存
$_SESSION['error'] = 'ログインに失敗しました。';

// input.php: 読み取って即削除
$error = $_SESSION['error'] ?? null;
unset($_SESSION['error']);  // ← これでリロードしても再表示されない
```

## ログイン後ページ（home/index.php）

保護ページは冒頭でセッションチェックを行います。

```php
require_once '../app.php';

$auth_user = AuthUser::check();

if (empty($auth_user)) {
    // ログインしていない場合はログインページへ強制リダイレクト
    header('Location: ../signin/');
    exit;
}
// ここより下はログイン済みユーザのみ到達できる
```

> `header()` のあとに `exit` を書かないと、リダイレクト後もスクリプトが続行され
> セキュリティホールになります。**必ず `exit` をセットで書く**ことが重要です。

## ログアウト（signout/index.php）

```php
require_once '../app.php';

AuthUser::clear();  // セッションからユーザ情報を削除

header('Location: ../signin/');
```

`AuthUser::clear()` の内部では `unset($_SESSION[SESSION_SECRET])` を実行しています。
セッションデータを完全に破棄する場合は `session_destroy()` を使うこともできます。

| 方法 | 動作 |
| ---- | ---- |
| `unset($_SESSION['key'])` | 特定のキーだけ削除 |
| `$_SESSION = []` | セッション変数をすべてクリア |
| `session_destroy()` | サーバのセッションファイルを削除 |

## セッションとクッキーの違い

| | セッション | クッキー |
| - | ----------- | ------- |
| 保存場所 | サーバ | ブラウザ（クライアント） |
| セキュリティ | 高い（IDだけブラウザに送る） | 低い（データがそのまま保存） |
| 有効期限 | ブラウザ終了 / `session_destroy()` | 任意に設定可能 |
| 用途 | ログイン状態・カート | 言語設定・テーマなど |

## まとめ

| 概念 | キーワード | 意味 |
| ---- | ---------- | ---- |
| セッション | `session_start()` / `$_SESSION` | サーバ側でページをまたいでデータを保持する仕組み |
| セッション固定攻撃対策 | `session_regenerate_id(true)` | ログイン時にセッションIDを再生成して攻撃を防ぐ |
| パスワード検証 | `password_verify()` | ハッシュ化されたパスワードと平文を安全に比較する |
| フラッシュメッセージ | `$_SESSION` + `unset()` | 1度だけ表示して消えるメッセージ |
| 認証チェック | `AuthUser::check()` | セッションにユーザ情報があるかを確認する |
| ログアウト | `AuthUser::clear()` | セッションからユーザ情報を削除する |
| 共通ファイル | `app.php` | セッション・定数・クラスのロードを1ファイルに集約 |
| モデル基底クラス | `Model` / `self::pdo()` | PDO接続を継承で使い回す設計パターン |
