## はじめに

PHPでは、`require` と `include` を使ってプログラムファイルを分割して管理することができます。コードを分割することで、「変更が1ヶ所で済む」「再利用できる」といった大きなメリットが得られます。

## なぜファイルを分けるのか

ファイルを役割ごとに分ける理由は、主に3つあります。

- **再利用性**: ヘッダーやフッター、共通の関数などを1つのファイルにまとめ、複数のページから読み込むことができます。
- **保守性**: 税率や共通設定の変更が必要な場合、`config.php` などのファイルを1ヶ所修正するだけで、サイト全体に反映されます。
- **可読性**: 1つのファイルが長くなりすぎるのを防ぎ、どこに何が書いてあるかを整理しやすくなります。

```php
// ❌ 分けない場合 — 全ページに同じコードをコピペ
const TAX_RATE = 0.10; // page1.php, page2.php, ... すべてに記述が必要

// ✅ 分けた場合 — config.php に1回書くだけ
require_once 'includes/config.php'; // どのページもこの1行でOK
```

## 4つの関数の違い

PHPにはファイルを読み込むための関数が4つあり、状況によって使い分けます。

| 関数 | ファイルがない場合 | 2回目以降の読み込み | 主な用途 |
| :--- | :--- | :--- | :--- |
| `require` | **Fatal Error（停止）** | 再度読み込む | DB接続など、動作に絶対必要なもの |
| `require_once` | **Fatal Error（停止）** | **スキップ（安全）** | 設定・定数ファイル、関数定義など |
| `include` | **Warning（続行）** | 再度読み込む | HTMLパーツ（ヘッダー・フッター）など |
| `include_once` | **Warning（続行）** | **スキップ（安全）** | 読み込み漏れを防ぎたいHTMLパーツなど |

> **使い分けの原則**
> そのファイルがなければプログラムが正常に動かない場合は `require_once` を使います。表示が欠けても致命的でないパーツには `include` を使います。迷った場合は、基本的には `require_once` を使うのが安全です。

## ファイル構成の実例

一般的なWebサイト制作での構成例です。

```text
├── index.php           ← メインページ
└── includes/
    ├── config.php      ← 定数・設定（require_once）
    ├── products.php    ← 商品データ（require_once）
    ├── header.php      ← 共通ヘッダー（include）
    └── footer.php      ← 共通フッター（include）
```

**index.php の冒頭での読み込み例**

```php
<?php
// 設定ファイル（必須） — なければ停止
require_once 'includes/config.php';

// 商品データ（必須） — なければ停止
require_once 'includes/products.php';

// ヘッダー（表示パーツ）— なければ警告が出るが表示は続く
include 'includes/header.php';
?>

<!-- ページ固有のコンテンツ -->

<?php
// フッター
include 'includes/footer.php';
?>
```

## スコープ（変数の受け渡し）

読み込まれたファイルは、その場所に**そのまま展開される**イメージです。そのため、読み込み元の変数は読み込まれたファイル内でもそのまま使用できます。

```php
// main.php
$username = "Yamada";
include 'greeting.php'; // ← この時点での $username が引き継がれる

// greeting.php
echo "こんにちは、{$username}さん！"; // → こんにちは、Yamadaさん！
```

> **注意点**
> 関数の中からファイルを読み込むと、その中での変数の有効範囲（スコープ）は関数の内部に限定されます。

## よくある使い方パターン

### パターン1：設定ファイル（config.php）

定数や環境設定を1ヶ所にまとめます。

```php
// config.php
const DB_HOST = 'localhost';
const DB_NAME = 'shop_db';
const TAX_RATE = 0.10;
```

### パターン2：共通レイアウトパーツ

ヘッダーやフッターなどの共通パーツを切り出します。

```php
// page.php
require_once 'config.php';

$pageTitle = "商品一覧";
include 'parts/header.php';  // header.php 内で $pageTitle を使える

// ページ固有のコンテンツ...

include 'parts/footer.php';
```

### パターン3：関数定義ファイル

自作の関数をまとめて読み込みます。

```php
// functions.php
function formatPrice($price) {
    return '¥' . number_format($price);
}

// 呼び出し側
require_once 'functions.php'; // 重複定義を防ぐために _once を使用
echo formatPrice(1980); // → ¥1,980
```
