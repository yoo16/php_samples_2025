## はじめに

このセクションでは、PHPプログラムを書くために必要な、最も基本的で重要なルールについて学びます。記号のひとつひとつに大切な意味があることを理解しましょう。

## PHPの基本構成

PHPプログラムは、専用のタグで囲むことで「ここからPHPが始まります」という合図を送ります。

### 開始タグと終了タグ

PHPのコードは、必ず `<?php` で書き始めます。

```php
<?php
echo "こんにちは！";
```

- **開始タグ**: `<?php`
- **終了タグ**: `?>`

> **注意：終了タグの省略**
> PHPのみで構成されるファイルの場合、終了タグ `?>` は省略するのが一般的です。これにより、ファイルの末尾に不要な空行やスペースが入ることで発生するエラー（「headers already sent」エラーなど）を防ぐことができます。

### HTMLへの埋め込み

PHPの最大の特徴は、**HTMLの中に直接埋め込める**ことです。

```html
<!DOCTYPE html>
<html>
<body>
    <h1><?php echo "タイトル"; ?></h1>
    <p>現在は <?= date('Y-m-d') ?> です。</p>
</body>
</html>
```

`<?= ... ?>` は `<?php echo ...; ?>` の **短縮形** で、HTMLの中に変数の値などを埋め込む際によく使われます。

## PHPの記述ルール

### 文末のセミコロン `;`

PHPでは、ひとつの命令が終わるごとに必ず **`;`（セミコロン）** をつけます。これは日本語の「。」（句点）のような役割です。

```php
echo "おはよう"; // セミコロンが必要
echo "おやすみ";
```

### 変数の目印 `$`

PHPでデータを一時的に保存する「変数」を扱うときは、必ず名前の前に **`$`（ドル記号）** をつけます。

```php
$message = "こんにちは、PHPの世界へ！";
echo $message;
```

### コメントの書き方

プログラムとして実行されない「メモ書き」を残すことができます。

```php
// 一行コメント：この行の末尾まで無視されます

/*
   複数行コメント
   長い説明を書くときに使います
*/

/**
 * DocBlockコメント
 * 関数やクラスの説明を書くための公式なスタイルです
 */
```

## プログラムの実行方法

PHPには大きく分けて2つの実行方法があります。

### 1. Webサーバー経由（ブラウザで表示）

`Apache` や `Nginx` などの **Webサーバー** を通じて実行します。
サーバーがプログラムを実行し、その **結果（HTML）だけ** がブラウザに届きます。

<img src="/storage/teaching_material/php_flow.svg" width="600" alt="PHPの実行フロー">

- **メリット**: 誰でもブラウザから利用できる。
- **ファイル例**: `demo.php`

### 2. CLI（コマンドラインインターフェース）

ターミナルやコマンドプロンプトから直接実行します。

```bash
php 01_hello/cli_demo.php
```

CLIでは、キーボードからの入力を受け取ったり、バッチ処理（自動化）を行ったりするのに適しています。

- **ファイル例**: `cli_demo.php`

## 環境の確認

PHPの現在の設定やバージョンを詳しく確認したいときは、`phpinfo()` という便利な命令があります。

```php
<?php
phpinfo();
```

> **セキュリティ上の注意**
> `phpinfo()` はサーバーの内部情報を詳細に表示します。本番環境（公開サーバー）に置いたままにすると攻撃のヒントを与えてしまう可能性があるため、確認が終わったら必ずファイルを削除しましょう。

## まとめ

- PHPは `<?php` で始める。
- 命令の終わりには `;` をつける。
- 変数には `$` をつける。
- HTMLに簡単に埋め込むことができる。
- ブラウザで動かす方法と、コマンドラインで動かす方法がある。
