## はじめに

プログラムの流れを「もし〜なら」という条件によって分岐させる、最も重要な制御構造について学びます。条件分岐をマスターすれば、ユーザーの状態に合わせた表示の切り替えなどが可能になります。

## if 文の基本とコロン構文

最も一般的な分岐です。HTML内に記述する場合は、`{ }` の代わりに `:` と `endif;` を使う「**コロン構文**」がよく使われます。

```php
<?php
// 通常の構文
if ($isAuth) {
    echo "ログイン中";
} else {
    echo "ゲスト";
}
?>

<!-- HTML内でのコロン構文（menu.phpなどで使用） -->
<?php if ($isAuth): ?>
    <li>マイページ</li>
<?php else: ?>
    <li>サインイン</li>
<?php endif; ?>
```

- **通常の構文**: PHPプログラムのみで完結する場合に適しています。
- **コロン構文**: HTMLとPHPが混在するテンプレートファイルで、どこまでが条件の範囲か見やすくするために使われます。

## 入れ子（ネスト）構造

if文の中にさらにif文を入れることで、複雑な条件を表現できます。これを「**入れ子（ネスト）**」と呼びます。

```php
<?php
if ($isMaintenance) {
    $message = "メンテナンスのため休止中";
} else {
    // メンテナンス中でない場合のみ、さらに判定を行う
    if ($charge >= $payment) {
        $message = "決済成功";
    } else {
        $message = "残高不足";
    }
}
?>
```

`payment.php` では、この構造を使って「メンテナンス中か？」を確認した後に「残高は足りているか？」を判定しています。

## match 式（PHP 8.0〜）

`switch` 文の進化版として導入されたのが `match` 式です。値を返すことができるため、変数への代入を非常にスッキリと記述できます。

```php
<?php
// 厳密な比較 (===) が行われ、breakも不要です
$garbage = match ($weekIndex) {
    1, 3    => "燃えるゴミ",
    5       => "燃えないゴミ",
    default => "回収なし",
};
?>
```

`garbage.php` では、曜日インデックスに応じたゴミの種類を判定するために活用しています。

> **Tips: switch と match の違い**
>
> - `switch` 文は「ゆるい比較 (`==`)」を行います。
> - `match` 式は「厳密な比較 (`===`)」を行います。
> 予期せぬ判定ミスを防ぐため、新しいPHPでは `match` 式の使用が推奨されます。

## まとめ

- **if 文**: 基本的な条件分岐。HTML内ではコロン構文が便利。
- **入れ子構造**: 条件の中にさらに条件を重ねる。
- **match 式**: 複数の値に対する分岐をスッキリ書ける。厳密な比較が行われる。
