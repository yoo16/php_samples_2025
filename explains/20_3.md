## 3：Like機能のコードを読む

### 講義（20分）

**フォームPOST vs AJAX の使い分け**

| 方式 | 特徴 | このアプリでの例 |
|---|---|---|
| フォームPOST | ページが遷移する | フォロー・削除 |
| AJAX（API） | ページが遷移しない | いいね・ツイート投稿 |

**Like機能の全体の流れ**

```
① ハートボタンをクリック（like.js）
    ↓ fetch POST
② api/like/update/index.php
    ↓ Like::update()
③ app/models/Like.php
    ↓ DBのlikesテーブルを更新
④ like_count と liked を JSON で返す
    ↓
⑤ like.js がボタンの色・数を更新
```

**コードを順番に読む**

1. `app/models/Like.php` → `update()` / `count()` / `fetch()`
2. `api/like/update/index.php` → 認証チェック・POST確認・JSON返却
3. `js/like.js` → fetch の書き方・レスポンスの受け取り方

**APIのルール（このアプリの設計思想）**

```php
header('Content-Type: application/json');  // 必ず先頭
// 未認証 → 401
// POST以外 → 405
// 入力不正 → 400
// 成功 → 200 + JSONデータ
```

## 4：仕様書から機能を実装する（Vibeコーディング）

### 講義（15分）

**仕様書駆動開発とは**

AIコーディングで重要なのは「何を作るか」を正確に伝えること。
仕様書があれば、AIは設計の意図を理解して実装できる。

**このアプリのVibe コーディングのルール**
- `CLAUDE.md` → プロジェクト全体の地図
- `SKILL.md` → 実装パターンの辞書
- `.claude/rules/` → 守るべきルール集
- `docs/` → 今回の作業指示（仕様書）

AIへの指示はシンプルに：
```
docs/reply_spec.md の仕様に基づいて返信機能を実装してください。
実装順序に従って1ステップずつ確認しながら進めてください。
```
