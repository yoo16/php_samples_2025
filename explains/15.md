## クラスの応用

前章ではクラス・インスタンス・継承の基本を学びました。
この章では、より実践的な設計として **インターフェース**・**抽象クラス**・**ポリモーフィズム** と、
ゲームロジックを分離する **サービスクラス** を学びます。

## プログラムの構成

```treeview
15_card_game/
├── interfaces/
│   └── CardInterface.php
├── models/
│   ├── BaseCard.php
│   ├── AquaCard.php

│   ├── KnightCard.php
│   └── ThunderCard.php
```

| `battle.php` | バトル画面、ボタン操作で GameService を呼ぶ |

## インターフェース (interface)

**インターフェース**は、クラスが「必ず実装しなければならないメソッド」を定義した**規約（契約）**です。
メソッドの名前・引数・返り値の型だけを宣言し、中身（処理）は書きません。

```php
// interfaces/CardInterface.php
interface CardInterface
{
    public function attack(BaseCard $target): int;
    public function specialSkill(BaseCard $target): int;
    public function gainExp(int $exp): void;
    public function isLevelUp(): bool;
    public function levelUp(): void;
}
```

### なぜインターフェースが必要か？

インターフェースがない場合、あるクラスに `attack()` があるかどうかは実装を見なければわかりません。
インターフェースを使うことで、

> **「このクラスは必ず `attack()` を持っている」** と保証される

ため、他のクラスから安全に呼び出せます。

---

## implements（インターフェースの実装）

インターフェースで定めた規約を「実装する」ことを `implements` と言います。

```php
// models/BaseCard.php
class BaseCard implements CardInterface
{
    // CardInterface で宣言したメソッドをすべて実装する義務がある
    public function attack(BaseCard $target): int { ... }
    public function specialSkill(BaseCard $target): int { ... }
    public function gainExp(int $exp): void { ... }
    public function isLevelUp(): bool { ... }
    public function levelUp(): void { ... }
}
```

`implements CardInterface` と書くことで、

- `CardInterface` で宣言されたメソッドを**すべて実装しなければコンパイルエラー**になる
- 外部から「このオブジェクトは `CardInterface` を満たしている」と信頼できる

という2つの恩恵が得られます。

## 抽象クラス (abstract class)

抽象クラスとは、**直接インスタンスを作ることができない「不完全なクラス」**です。
継承して使うことを前提として設計します。

```php
abstract class BaseCard implements CardInterface
{
    // 共通ロジックはここに書く

    // abstract メソッド: 子クラスに実装を強制する
    abstract public function specialSkill(BaseCard $target): int;
}
```

```php
$card = new BaseCard(...); // ❌ エラー：抽象クラスは直接 new できない
$card = new AquaCard();    // ✅ 子クラスなら OK
```

今回の `15_card_game` では `BaseCard` は通常クラスとして実装していますが、
「`BaseCard` 単体では意味を持たず、必ず具体的なカードとして使う」という設計にしたい場合は
`abstract` を付けると明確になります。

| キーワード | 意味 |
| ----------- | ------------------------------ |
| `interface` | メソッドの名前・型だけ定義（処理なし） |
| `abstract class` | 一部未実装のクラス（直接 new 不可） |
| `extends` | クラスを継承する |
| `implements` | インターフェースを実装する |

## ポリモーフィズム（多態性）

**ポリモーフィズム**とは、異なるクラスのインスタンスを**同じ操作で扱える**性質のことです。

`card_list.php` では、4種類のカードを `foreach` でまとめて処理しています。

```php
// 4種類のカードのインスタンスを配列にまとめる
$cards = [
    'aqua'    => new AquaCard(),
    'knight'  => new KnightCard(),
    'forest'  => new ForestCard(),
    'thunder' => new ThunderCard(),
];

// クラスの違いを気にせず、同じコードで扱える
foreach ($cards as $id => $card) {
    // $card が AquaCard でも ThunderCard でも、同じプロパティ・メソッドが使える
    echo $card->name;
    echo $card->attack;
    include 'views/card.php';
}
```

それぞれのクラスが異なっても、共通の `BaseCard` から継承しているため、
同じプロパティ・メソッドを持っていることが保証されます。

### ポリモーフィズムのメリット

もし新しいカード `FireCard` を追加したい場合、`foreach` の中身は一切変更不要です。

```php
// 追加するだけで自動的に一覧に表示される
$cards = [
    'aqua'    => new AquaCard(),
    'knight'  => new KnightCard(),
    'forest'  => new ForestCard(),
    'thunder' => new ThunderCard(),
    'fire'    => new FireCard(), // ← これを追加するだけ
];
```

## サービスクラス (GameService)

**サービスクラス**とは、ゲームのロジック（処理）だけをまとめたクラスです。
`battle.php`（画面）と `GameService`（処理）を分けることで、それぞれの責任が明確になります。

### プロパティ

```php
class GameService
{
    public ?BaseCard $player = null;  // プレイヤーカード（未選択なら null）
    public ?BaseCard $enemy  = null;  // 敵カード（未出現なら null）
    public string $message = "アクションを選択してください。";
```

`?BaseCard` の `?` は **nullable（null を許容する）** という意味です。
カードがまだ選ばれていない状態を `null` で表します。

### コンストラクタ：セッションからの復元

```php
public function __construct()
{
    if (isset($_SESSION['player_card'])) {
        $this->player = $_SESSION['player_card'];
    }
    if (isset($_SESSION['enemy_card'])) {
        $this->enemy = $_SESSION['enemy_card'];
    }
}
```

`new GameService()` した瞬間に、セッションに保存されていたカードを復元します。
これにより、ページをリロードしても前回のカード状態が引き継がれます。

## GameService の各メソッド

### setupPlayer() ― プレイヤーカードの初期化

`card_list.php` から送られてきたカード ID を受け取り、対応するカードのインスタンスを生成します。
`match` 式で ID と子クラスを対応させ、セッションに保存しています。

```php
public function setupPlayer(string $cardId): void
{
    $this->player = match ($cardId) {
        'aqua'    => new AquaCard(),
        'knight'  => new KnightCard(),
        'forest'  => new ForestCard(),
        'thunder' => new ThunderCard(),
        default   => $this->player, // 不明なIDなら現状維持
    };
    $_SESSION['player_card'] = $this->player; // セッションに保存
}
```

`match` 式は `switch` と似ていますが、値を直接返せるため変数への代入がシンプルに書けます。

### setupEnemy() ― 敵カードのランダム生成

敵カードがまだいない場合のみ、4種類の中からランダムで1枚を選びます。
すでに存在する場合（ページリロード時など）は何もしないため、対戦相手が変わりません。

```php
public function setupEnemy(): void
{
    if (!$this->enemy) { // 敵がいない場合のみ実行
        $enemy_cards = [new AquaCard(), new ForestCard(), new KnightCard(), new ThunderCard()];
        $this->enemy = $enemy_cards[array_rand($enemy_cards)];
        $_SESSION['enemy_card'] = $this->enemy;
    }
}
```

### handleAction() ― アクションの振り分け（公開API）

ボタンで送られてきたアクション名を受け取り、対応するプライベートメソッドに振り分けます。
このメソッドだけが `public` で、外部（`battle.php`）から呼べる唯一の入口です。

```php
public function handleAction(string $action): void
{
    $this->message = match ($action) {
        'attack'   => $this->attack(),
        'special'  => $this->special(),
        'gain_exp' => $this->gainExp(),
        'reset'    => $this->reset(),
        default    => $this->message,
    };
}
```

### attack() ― 通常攻撃

プレイヤーカードの `attack()` メソッドを呼び、与えたダメージをメッセージに整形して返します。

```php
private function attack(): string
{
    $dmg = $this->player->attack($this->enemy);
    return "{$this->player->name} の攻撃！\n{$this->enemy->name} に {$dmg} のダメージを与えた！";
}
```

### special() ― 必殺技

MP が残っている場合のみ必殺技を発動します。MP が 0 なら発動せずにメッセージを返します。

```php
private function special(): string
{
    if ($this->player->mp > 0) {
        $dmg = $this->player->specialSkill($this->enemy);
        return "{$this->player->name} の必殺技「{$this->player->specialSkill}」！\n{$this->enemy->name} に {$dmg} のダメージを与えた！";
    }
    return "MPが足りません！";
}
```

### gainExp() ― 経験値獲得とレベルアップ

20〜50 のランダムな経験値を付与し、レベルアップ条件を満たしていれば自動でレベルアップします。

```php
private function gainExp(): string
{
    $exp = rand(20, 50);
    $this->player->gainExp($exp);
    $msg = "{$this->player->name} は {$exp} の経験値を獲得した！";
    if ($this->player->isLevelUp()) {
        $this->player->levelUp();
        $msg .= "\nレベルアップ！ LV.{$this->player->level} になった！";
    }
    return $msg;
}
```

### reset() ― リセット

セッションからカード情報を削除してカード選択画面に戻します。

```php
private function reset(): string
{
    unset($_SESSION['player_card']);
    unset($_SESSION['enemy_card']);
    header("Location: card_list.php");
    exit;
}
```

## public / private（アクセス修飾子）

`GameService` のメソッドには `public` と `private` の2種類があります。

| 修飾子 | 意味 | このクラスでの用途 |
| ------- | ------------------------------ | ------------------------- |
| `public` | クラスの外からも呼べる | `handleAction`、`setupPlayer`、`setupEnemy` |
| `private` | クラスの内部からしか呼べない | `attack`、`special`、`gainExp`、`reset` |

`attack()` などを `private` にすることで、外部から直接呼ばれることを防いでいます。
外部から触れるのは `handleAction()` だけ ― これが**公開インターフェース（API）**の役割を果たします。

## メインプログラムとの分離

`battle.php` と `GameService` がそれぞれ何を担当するかを整理します。

| | `battle.php` | `GameService` |
|--|--|--|
| **役割** | 表示・入出力 | ゲームロジック |
| **やること** | HTTPリクエスト受付、HTML出力 | 攻撃計算、経験値処理、セッション管理 |
| **知っていること** | `$game` というオブジェクトの存在 | カードのステータスと状態 |
| **知らなくてよいこと** | ダメージ計算式・レベルアップ条件 | HTMLの構造・ボタンの見た目 |

`battle.php` はロジックを一切持たず、すべてを `GameService` に委ねます。

```php
// battle.php の処理部分（HTMLは一切書かない）

require_once __DIR__ . '/services/GameService.php';
session_start();

$game = new GameService();            // ① セッションからカードを復元

if ($cardId = $_GET['card_id'] ?? '') {
    $game->setupPlayer($cardId);      // ② カード選択があれば初期化
}

if (!$game->player) {
    header('Location: card_list.php'); // ③ カード未選択なら選択画面へ
    exit;
}

$game->setupEnemy();                  // ④ 敵がいなければランダム生成

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $game->handleAction($_POST['action'] ?? ''); // ⑤ ボタン操作を処理
}

// ⑥ 表示用に変数を渡すだけ
$player  = $game->player;
$enemy   = $game->enemy;
$message = $game->message;
```

### 分離する前後の比較

**分離しない場合（ロジックが battle.php に混在）**

```php
// battle.php の中にロジックが混ざっている ❌
if ($_POST['action'] === 'attack') {
    $baseDmg = ($player->attack * 1.5) - $enemy->defense;
    $dmg = (int)($baseDmg + rand(-5, 5));
    if ($dmg < $player->attack * 0.2) $dmg = (int)($player->attack * 0.2);
    $enemy->hp -= $dmg;
    $message = "{$player->name} の攻撃！ {$dmg} のダメージ！";
}
// さらに HTML が続く...
```

**分離した場合（battle.php はシンプル）**

```php
// battle.php はメソッドを呼ぶだけ ✅
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $game->handleAction($_POST['action'] ?? '');
}
```

ロジックが `GameService` に集まることで、

- `battle.php` は読みやすくなる
- ダメージ計算式を変えたいときは `GameService` だけ修正すればよい
- 将来 `api.php` を作るときも同じ `GameService` を再利用できる

## battle.php の全体の流れ

```
[card_list.php]
  ↓ フォームで card_id を送信（GET）
[battle.php]
  ① new GameService()       → セッションからカードを復元
  ② setupPlayer($cardId)    → カードIDからインスタンスを生成・保存
  ③ setupEnemy()            → 敵をランダム生成（初回のみ）
  ④ handleAction($action)   → POST ボタンの処理を振り分け
      "attack"   → attack()   → player->attack(enemy)
      "special"  → special()  → player->specialSkill(enemy)
      "gain_exp" → gainExp()  → gainExp() → isLevelUp() → levelUp()
      "reset"    → reset()    → セッション削除 → card_list.php へ
  ⑤ $player, $enemy, $message を HTML に渡して表示
```

## まとめ

| 概念 | キーワード | 意味 |
| -------------- | ------------- | ------------------------------------------ |
| インターフェース | `interface` | メソッドの規約を定義する（処理なし） |
| 実装 | `implements` | インターフェースの規約を満たすと宣言する |
| 抽象クラス | `abstract` | 直接 new できない・継承専用のクラス |
| 継承 | `extends` | 親クラスの機能を引き継ぐ |
| ポリモーフィズム | — | 異なるクラスを同じ操作で扱える性質 |
| サービスクラス | — | ロジックをまとめた役割分担のクラス |
| match 式 | `match` | 値に応じて処理を分岐し、結果を返す |
