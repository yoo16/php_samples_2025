# 教材作成スタイルガイド

## 目的

このガイドは、`Web`システム開発教材の `Markdown` 原稿を、統一した書き方で作成するためのルールをまとめたものです。

---

## 基本方針

- 初学者にわかりやすい順番で説明する
- 専門用語は必要に応じて補足する
- 文章だけで伝わりにくい内容は表や図を使う
- 章ごとに表記ルールを統一する

---

## 見出しルール

- `##` を章内の主要見出しとして使う
- `###` を小見出しとして使う
- 見出しの中では英単語や技術用語を `` ` ` `` で囲まない

### 例

```md
## Web APIとは

### APIとは
```

---

## 本文ルール

- 普通の文章では、英単語や技術用語をある程度 `` ` ` `` で囲む
- ただし、過剰に囲みすぎない
- 初学者が重要点を見つけやすいように、強調したい文章は `** **` を使う

### 例

```md
`Web API` は `HTTP` を使ってデータをやり取りする `API` です。

**フロントエンドとバックエンドをつなぐ役割**があります。
```

---

## 表のルール

- 表の中では `` ` ` `` を使わない
- 比較、整理、一覧化した方がわかりやすい内容は表にする
- 表は `| |` 形式で記述する

### 例

```md
| 項目 | 内容 |
| ---- | ---- |
| Web API | HTTP を使ってデータをやり取りする仕組み |
| JSON | API でよく使われるデータ形式 |
```

---

## 補足と注意書き

- `Note` や補足説明、注意点は `>` を使う
- 補足は短く、要点だけを書く

### 例

```md
> Web API は、画面そのものではなく、データや処理結果を返すために使われることが多いです。
```

---

## 章末のルール

- 章末見出しは `## まとめ` で統一する
- `### この章のまとめ` は使わない
- まとめは表形式を基本とする

### 例

```md
## まとめ

| 項目 | 内容 |
| ---- | ---- |
| Web API | HTTP を使ってデータをやり取りする仕組み |
| JSON | API でよく使われるデータ形式 |
```

---

## 図と画像のルール

- フロー図は `svg` を利用する
- 画面キャプチャーは `png` などを利用する
- 図は文字や矢印が重ならないようにシンプルにする
- 矢印は必要最小限にする

### 画像記述例

```html
<img src="/storage/teaching_material/web_api_overview.svg" width="500">
```

---

## 教材構成ルール

- `Markdown` ファイル単位で教材を管理する
- `##` が `TOC` のタイトルになる前提で構成する
- `###` 以降を各節の小見出しとして使う
- `##` はあまり数を増やさないよう、見出し3以降で調整する

---

## 文体ルール

- 文体は「です・ます調」で統一する
- 初学者向けに、断定しすぎずわかりやすく説明する
- 難しい内容は、まず概要を説明してから詳細に入る

---

## 推奨する説明順

- まず概要を説明する
- 次に役割や仕組みを説明する
- その後に具体例を示す
- 最後に表やまとめで整理する

---

## 用語の扱い

- 本文では `Web`、`HTTP`、`API`、`JSON`、`HTML`、`CSS`、`JavaScript` などを適度に `` ` ` `` で囲む
- 見出しでは囲まない
- 表では囲まない

---

## このガイドの使い方

- 新しい教材を作るときは、このルールに従って記述する
- 既存教材を修正するときも、できるだけこの形式に寄せる
- 表記ゆれが出た場合は、このガイドを優先する
