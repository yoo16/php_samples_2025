## はじめに

複数のデータをまとめて扱う **配列（Array）** は、Web制作で最も頻繁に使うデータ形式です。ユーザー一覧や商品リストのように「同じ種類のデータの集まり」を表現するのに適しています。インデックス配列・連想配列・多次元配列の違いと、データの検索・操作方法を実例で学びましょう。

## インデックス配列

番号（0, 1, 2...）でデータを管理する、最もシンプルな配列です。`[]` を使って定義し、**インデックスは必ず 0 から始まります**。

```php
<?php
$drinks = ["コーラ", "お茶", "水"];

echo $drinks[0]; // コーラ
echo $drinks[1]; // お茶
echo $drinks[2]; // 水

// 要素の数を調べる
echo count($drinks); // 3

// 末尾に追加する
$drinks[] = "コーヒー";
// または: array_push($drinks, "コーヒー");
```

> **注意**
> インデックスは `0` から始まります。3つの要素がある場合、最後のインデックスは `2`（3 - 1）です。`$drinks[3]` にアクセスすると未定義エラーになります。

## 連想配列（キーと値のペア）

番号の代わりに「名前（**キー**）」をつけてデータを管理する配列です。`"キー" => 値` の形式で記述します。`users.php` のユーザーデータはこの形式です。

```php
<?php
$user = [
    "id"           => 1,
    "account_name" => "chris",
    "display_name" => "Chris Johnson",
    "email"        => "chris@example.com",
    "role"         => "Administrator",
    "bio"          => "Webデザイナー兼開発者。UI/UXデザインとPHPが得意です。",
    "avatar"       => "1.jpg",
];

// キー名でアクセスする
echo $user["display_name"]; // Chris Johnson
echo $user["role"];         // Administrator
```

- **インデックス配列**: `$arr[0]`, `$arr[1]` — 順番が重要なリストに使う
- **連想配列**: `$arr["name"]`, `$arr["email"]` — 意味を持ったキーでアクセスしたいときに使う

## 多次元配列（配列の入れ子）

「**配列の中に配列**」を入れることで、表形式のような複雑なデータ構造を表現できます。`index.php` でユーザー一覧を表示しているデータはこの形式です。

```php
<?php
$users = [
    [
        "id"           => 1,
        "account_name" => "chris",
        "display_name" => "Chris Johnson",
        "role"         => "Administrator",
    ],
    [
        "id"           => 2,
        "account_name" => "bob",
        "display_name" => "Bob Smith",
        "role"         => "Editor",
    ],
    [
        "id"           => 3,
        "account_name" => "alice",
        "display_name" => "Alice Johnson",
        "role"         => "User",
    ],
];

// [行インデックス]["キー"] でアクセス
echo $users[0]["display_name"]; // Chris Johnson
echo $users[1]["role"];         // Editor
```

## 外部ファイルからのデータ読み込み

データと表示ロジックを分離するために、配列データを別ファイルに切り出して `require_once` で読み込むパターンをよく使います。

```php
<?php
// index.php や detail.php の先頭で読み込む
require_once __DIR__ . '/data/users.php';
// → $users 配列がこのファイルで使えるようになる

// isset() でデータの存在確認をしてから描画
if (isset($users) && is_array($users)) {
    foreach ($users as $user) {
        echo $user['display_name'];
    }
}
```

- **`require_once`**: ファイルを1回だけ読み込む。ファイルが存在しない場合はエラーで停止。
- **`__DIR__`**: 現在のファイルが存在するディレクトリの絶対パス。パスのズレを防げる。

## 便利な配列関数

PHPには配列を操作するための関数が豊富に用意されています。

| 関数 | 用途 |
| :--- | :--- |
| `count($array)` | 要素の数を返す |
| `isset($array["key"])` | キーが存在するか確認 |
| `array_push($array, $item)` | 末尾に要素を追加（`$array[] = $item` と同じ） |
| `explode(",", $string)` | 文字列を区切り文字で分解して配列にする |
| `implode(",", $array)` | 配列を文字列に結合する（`explode` の逆） |
| `array_column($array, "key")` | 多次元配列から特定キーの値だけを取り出す |

```php
<?php
$users = [/* ... */];

// 件数表示
echo count($users) . '件のユーザー';

// 特定キーの値だけを一覧で取り出す
$ids    = array_column($users, 'id');    // [1, 2, 3]
$names  = array_column($users, 'display_name'); // ["Chris Johnson", ...]

// CSV文字列を配列に変換
$csv   = "PHP,JavaScript,Python";
$langs = explode(",", $csv); // ["PHP", "JavaScript", "Python"]
```

## 多次元配列からのデータ検索

`detail.php` では URL パラメータで受け取った `id` をもとに、`$users` 配列から該当ユーザーを検索しています。主に3つの方法があります。

### foreach で検索（基本）

最も標準的で、どの言語でも共通の考え方です。見つかったら `break` でループを抜けるのがポイントです。

```php
<?php
$id   = (int)$_GET['id'];
$user = null;

foreach ($users as $u) {
    if ($u['id'] === $id) {
        $user = $u;
        break; // 見つかったらループを終了
    }
}
```

### array_filter() で検索（モダン）

条件に合う要素だけを抽出します。PHP 7.4以降の**アロー関数**（`fn()`）を使うと簡潔に書けます。

```php
<?php
// array_filter：条件に合う要素だけを残す
$result = array_filter($users, fn($u) => $u['id'] === $id);

// reset()：配列の先頭要素を取り出す（filter結果はインデックスが保持される）
$user = $result ? reset($result) : null;
```

### array_column() + array_search() で検索

`array_column` で id のみの配列を作り、`array_search` でインデックスを特定します。一行で書けるコンパクトな手法です。

```php
<?php
// array_column($users, 'id') → [1, 2, 3]
// array_search($id, ...) → 一致したインデックスを返す
$key  = array_search($id, array_column($users, 'id'), true);
$user = ($key !== false) ? $users[$key] : null;
```

> **見つからなかった場合の処理**
> どの手法でも `$user` が `null` になる可能性があります。`detail.php` では `if (!$user)` でチェックし、`header("Location: ./")` で一覧ページにリダイレクトしています。

## まとめ

- **インデックス配列** — 順番があるリストに使う。インデックスは `0` から始まる。
- **連想配列** — キー名でアクセスするデータ構造。`"key" => value` の形式で記述。
- **多次元配列** — 配列の中に配列を入れてテーブル形式のデータを表現。
- データは `require_once` で外部ファイルに分離すると、保守しやすくなる。
- 配列からの検索は `foreach` + `break`、`array_filter()`、`array_column()` + `array_search()` の3通りがある。
