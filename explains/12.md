## はじめに

ユーザーからのデータ送信（**GET/POST**）と、ページをまたいで情報を保持する**セッション**の仕組みを、検索シミュレーター（`get_request.php`）と認証シミュレーター（`post_request.php`）の実例で学びましょう。

## GET と POST の違い

フォームからデータを送る方法には主に2つあります。`method` 属性で指定し、用途に合わせて使い分けることが重要です。

| | GET | POST |
| :--- | :--- | :--- |
| **データの場所** | URLの末尾（`?key=value`） | リクエストボディ（URL非表示） |
| **向いている用途** | 検索・フィルター・ページ番号 | ログイン・個人情報・大容量データ |
| **ブックマーク** | ✅ できる | ❌ できない |
| **機密情報** | ❌ 不向き | ✅ 向いている |

```php
<!-- GET：URLに ?keyword=PHP&category=Web が付く -->
<form action="" method="get">
    <input type="text" name="keyword" value="PHP">
    <input type="text" name="category" value="Web">
    <button type="submit">検索</button>
</form>

<!-- POST：URLにデータは出ない -->
<form action="post_receive.php" method="post">
    <input type="text"     name="email"    value="user@example.com">
    <input type="password" name="password" value="...">
    <button type="submit">ログイン</button>
</form>
```

> **XSS対策**
> フォームの値を画面に表示するときは、必ず `htmlspecialchars()` でエスケープしましょう。`<` や `>` などの特殊文字を無害な文字列に変換し、スクリプト埋め込み（XSS）を防ぎます。

## GETリクエストの受け取り

`$_GET` は送信されたすべての GETパラメータを連想配列として保持する**スーパーグローバル変数**です。`get_request.php` ではフォームから受け取ったデータを画面に表示しています。

```php
<?php
// $_GET にすべての GETパラメータが入る
// URL: get_request.php?keyword=PHP&category=Web の場合
$queries  = $_GET;

// ?? 演算子でキーが存在しない場合のデフォルト値を設定
$keyword  = $queries['keyword']  ?? '';
$category = $queries['category'] ?? '';

// 画面への出力は必ず htmlspecialchars() でエスケープ
echo htmlspecialchars($keyword); // "PHP"

// $queries が空か確認して出し分け
if (empty($queries)) {
    // まだ送信されていない
} else {
    // $queries をループして受信データを一覧表示
    foreach ($queries as $key => $value) {
        echo htmlspecialchars($key) . ': ' . htmlspecialchars($value);
    }
}

// $_SERVER['REQUEST_URI'] で現在のURLパス＋クエリ文字列を取得
echo $_SERVER['REQUEST_URI']; // "/12_form_session/get_request.php?keyword=PHP&..."
```

- `$_GET` は送信前（URLにパラメータなし）は空の配列 `[]` です。
- `empty($_GET)` で「まだ送信されていない」状態を判定できます。

## セッションによる状態保持

HTTPは本来「一回きり」で情報を忘れますが、`$_SESSION` を使うと**ページを移動してもデータを保持**できます。`post_request.php` では、ログイン状態とフラッシュメッセージをセッションで管理しています。

```php
<?php
// セッションを使うすべてのファイルの先頭で必ず呼ぶ
session_start();

// セッションからデータを取得（なければデフォルト値）
$posts    = $_SESSION['previous_post'] ?? [];
$status   = $_SESSION['status']        ?? '';
$authUser = $_SESSION['authUser']      ?? null;

// フラッシュメッセージ：1回表示したら削除する
function flashMessage(): string
{
    if (isset($_SESSION['message'])) {
        $message = $_SESSION['message'];
        unset($_SESSION['message']); // 取り出したら即削除
        return $message;
    }
    return '';
}

$message = flashMessage();

// status は表示後に削除（リロードで再表示されないように）
unset($_SESSION['status']);
```

> **フラッシュメッセージ**
> 「ログインに成功しました」のように**1回だけ表示して消えるメッセージ**をフラッシュメッセージといいます。セッションに保存して、読み取った直後に `unset()` するパターンが定番です。

## POSTの受け取りとPRGパターン

`post_receive.php` は POST データを受け取り、認証チェックを行ってからリダイレクトします。この **Post/Redirect/Get（PRGパターン）** により、ブラウザの再読み込みでフォームが二重送信されるのを防ぎます。

```php
<?php
session_start();

// テストユーザー情報（実際はデータベースで管理する）
const TEST_USER = [
    'name'     => '東京 太郎',
    'email'    => 'user@example.com',
    'password' => 'password123'
];

if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $email    = $_POST['email']    ?? '';
    $password = $_POST['password'] ?? '';

    // 入力値をセッションに保存（フォームの復元用）
    $_SESSION['previous_post'] = $_POST;

    // 認証検証
    if ($email === TEST_USER['email'] && $password === TEST_USER['password']) {
        // 認証成功
        $_SESSION['status']   = 'success';
        $_SESSION['authUser'] = TEST_USER;
        $_SESSION['message']  = 'ログインに成功しました。';

        // パスワードはセッションに残さない（セキュリティの基本）
        unset($_SESSION['previous_post']['password']);
    } else {
        // 認証失敗
        $_SESSION['status']  = 'error';
        $_SESSION['message'] = 'メールアドレスまたはパスワードが正しくありません。';
    }
}

// PRGパターン：処理後は必ずリダイレクト
header('Location: post_request.php');
exit; // header() の後は必ず exit でスクリプトを終了する
```

PRGパターンの流れは次のとおりです。

| ステップ | 処理 | ファイル |
| :--- | :--- | :--- |
| **Post** | フォームを POST 送信 | `post_request.php` |
| **Redirect** | 受信・検証後にリダイレクト | `post_receive.php` |
| **Get** | セッションを読んで結果を表示 | `post_request.php` |

> **なぜ exit が必要か**
> `header()` はレスポンスヘッダを設定するだけで、スクリプトの実行は続きます。`exit` を書かないと、リダイレクト後も後続のコードが実行されてしまいます。

## セッションのクリア（ログアウト）

ログアウト処理は `logout.php` で行います。`unset()` で特定のキーを削除してから、ログインページへリダイレクトします。

```php
<?php
session_start();

// 特定のキーだけ削除（必要なセッションは残せる）
if (isset($_SESSION['authUser'])) {
    unset($_SESSION['authUser']);
    unset($_SESSION['message']);
    unset($_SESSION['status']);
}

// ログインページへリダイレクト
header('Location: post_request.php');
exit;
```

| 方法 | 用途 |
| :--- | :--- |
| `unset($_SESSION['key'])` | 特定のキーだけ削除。他のデータは残る。ログアウトや個別データの削除に使う |
| `$_SESSION = []` + `session_destroy()` | すべてのデータを空にしてセッション自体を破棄。完全な初期化に使う |

## まとめ

- **GET** — URLにパラメータを付加して送る。検索や共有に向く。`$_GET` で受け取る。
- **POST** — リクエストボディに隠して送る。ログインや機密情報に向く。`$_POST` で受け取る。
- **`session_start()`** — セッションを使うすべてのファイルの先頭で必ず呼ぶ。
- **`$_SESSION`** — ページをまたいでデータを保持するスーパーグローバル変数。
- **フラッシュメッセージ** — セッションに保存して1回表示したら `unset()` するパターン。
- **PRGパターン** — POST 受信後にリダイレクトして GET で表示。二重送信を防ぐ。
- **`header()` + `exit`** — リダイレクト後は必ず `exit` でスクリプトを停止する。
