## ログインフォーム

`Web` アプリケーションでは、登録済みユーザが自分のアカウントで再び利用できるように、`ログイン機能` を用意します。  
この章では `19_signin/` を題材に、ログインフォームの表示、認証処理、ログイン後ページへの移動、ログアウトまでの流れを学びます。

| 項目 | 内容 |
| ---- | ---- |
| ログインフォーム | アカウント名とパスワードを入力する画面 |
| 認証 | 入力内容と `DB` のユーザ情報を照合する処理 |
| ログイン状態 | 認証成功後のユーザ情報をセッションに保持した状態 |
| ログアウト | セッションを削除してログイン状態を終了する処理 |

> `18.md` で説明した `セッション` や `app.php`、`Model` 基底クラスの考え方は、ここでも同じです。

## ログイン処理の全体像

ログイン機能は、次の順番で動きます。

| 手順 | 内容 |
| ---- | ---- |
| 1 | `signin/input/` でログインフォームを表示する |
| 2 | ユーザがアカウント名とパスワードを入力して送信する |
| 3 | `signin/auth/` で入力値を受け取る |
| 4 | `User::auth()` で認証を行う |
| 5 | 成功時は `AuthUser::set()` でセッションへ保存する |
| 6 | `home/` に移動してログイン後ページを表示する |
| 7 | `signout/` でログアウトし、セッションを削除する |

## ログインフォームの表示

`signin/input/index.php` では、ログインフォームを表示しています。  
このページでは、前回の入力値とエラーメッセージも取り出しています。

```php
$form = $_SESSION['signin'] ?? null;

$error = $_SESSION['error'] ?? null;
unset($_SESSION['error']);
```

### 入力値を保持する理由

認証に失敗したとき、アカウント名をもう一度入力しなくて済むようにするためです。

```php
<input type="text" name="account_name" value="<?= $form['account_name'] ?? '' ?>">
```

### エラー表示の流れ

| 処理 | 内容 |
| ---- | ---- |
| 保存 | 認証失敗時に `$_SESSION['error']` にメッセージを入れる |
| 表示 | `input/index.php` で取り出して表示する |
| 削除 | 表示後に `unset()` して残らないようにする |

> パスワードは再表示しないのが一般的です。

---

## signin/index.php の役割

`signin/index.php` は、直接フォームを書くページではなく、入力画面へ移動させるための入口です。

```php
header('Location: ./input/');
```

このようにしておくと、`signin/` にアクセスしたときの移動先を1か所で決められます。

| 利点 | 内容 |
| ---- | ---- |
| 入口をそろえられる | 利用者は `signin/` にアクセスすればよい |
| 構成を変えやすい | 将来の画面変更時も移動先を調整しやすい |

## 認証処理

`signin/auth/index.php` が、ログイン処理の中心です。

```php
if ($_SERVER['REQUEST_METHOD'] !== 'POST') {
    exit;
}

$_SESSION['signin'] = $_POST;

$account_name = $_POST['account_name'];
$password = $_POST['password'];

$user = new User();
$auth_user = $user->auth($account_name, $password);
```

### 処理のポイント

| ポイント | 内容 |
| ---- | ---- |
| `POST` のみ受け付ける | `GET` で認証処理が実行されないようにする |
| 入力値をセッションへ保存する | 失敗時にフォームへ再表示できる |
| `User::auth()` を呼ぶ | 認証の実処理をモデルへ分離する |

---

## User::auth() の役割

認証の本体は `User` モデルの `auth()` メソッドです。

```php
public function auth($account_name, $password)
{
    $pdo = self::pdo();
    $sql = "SELECT * FROM users WHERE account_name = :account_name";

    $stmt = $pdo->prepare($sql);
    $stmt->execute(['account_name' => $account_name]);
    $user = $stmt->fetch(PDO::FETCH_ASSOC);

    if ($user && password_verify($password, $user['password'])) {
        return $user;
    }
    return;
}
```

### auth() の流れ

| 手順 | 内容 |
| ---- | ---- |
| 1 | アカウント名でユーザを検索する |
| 2 | 該当ユーザがあるか確認する |
| 3 | `password_verify()` でパスワードを照合する |
| 4 | 一致すればユーザ情報を返す |
| 5 | 一致しなければ `null` を返す |

### なぜ account_name で検索するのか

この教材では、ログイン時の識別子として `アカウント名` を使っています。

| 識別子 | 用途 |
| ---- | ---- |
| account_name | ログイン時のユーザ識別 |
| password | 本人確認 |

---

## password_verify() の役割

ログインでは、入力されたパスワードと `DB` に保存されたハッシュ値を比較します。  
そのために使うのが `password_verify()` です。

```php
if ($user && password_verify($password, $user['password'])) {
    return $user;
}
```

### password_verify() の特徴

| 項目 | 内容 |
| ---- | ---- |
| 比較対象 | 入力された平文パスワードと保存済みハッシュ |
| 戻り値 | 一致すれば `true`、不一致なら `false` |
| 用途 | ログイン認証 |

> 登録時は `password_hash()`、ログイン時は `password_verify()` という組み合わせで使います。

## 認証成功と失敗の分岐

`signin/auth/index.php` では、認証結果によって処理を分けています。

```php
if (empty($auth_user['id'])) {
    $_SESSION['error'] = 'アカウント名またはパスワードが間違っています。';
    header('Location: ../input/');
    exit;
} else {
    AuthUser::set($auth_user);
    header('Location: ../../home/');
    exit;
}
```

### 分岐の意味

| 結果 | 処理 |
| ---- | ---- |
| 失敗 | エラーメッセージを保存して入力画面へ戻る |
| 成功 | ログインユーザを保存して `home/` へ移動する |

## AuthUser クラス

ログインユーザの扱いは `AuthUser` クラスにまとめています。

```php
class AuthUser extends User
{
    public static function check()
    {
        if (isset($_SESSION[self::$key])) {
            return $_SESSION[self::$key];
        }
    }

    public static function set($user)
    {
        $_SESSION[self::$key] = $user;
    }

    public static function clear()
    {
        if (isset($_SESSION[self::$key])) {
            unset($_SESSION[self::$key]);
        }
    }
}
```

### メソッドの役割

| メソッド | 役割 | 使う場所 |
| ---- | ---- | ---- |
| `set()` | ログインユーザを保存する | `signin/auth/index.php` |
| `check()` | ログイン中か確認する | `home/index.php` |
| `clear()` | ログイン情報を削除する | `signout/index.php` |

---

## ログイン後ページのチェック

`home/index.php` では、最初にログイン確認をしています。

```php
$auth_user = AuthUser::check();

if (empty($auth_user)) {
    header('Location: ../signin/');
    exit;
}
```

ログインしていない状態で `home/` に直接アクセスしても、ログイン画面へ戻されます。

### ログインチェックの目的

| 目的 | 内容 |
| ---- | ---- |
| 非ログイン時のアクセスを防ぐ | 保護ページをそのまま見せない |
| 認証済みページを分ける | ログイン前とログイン後の画面を区別できる |

---

## ログアウト処理

`signout/index.php` では、ログイン情報を削除してからログイン画面へ戻します。

```php
use App\Models\AuthUser;

AuthUser::clear();
header('Location: ../signin/');
```

### ログアウトの流れ

| 手順 | 内容 |
| ---- | ---- |
| 1 | `AuthUser::clear()` でセッションを削除する |
| 2 | `signin/` にリダイレクトする |

## ログインフォームで学ぶポイント

この章の中心は、`フォーム表示` と `認証処理` の分担です。

| 観点 | 内容 |
| ---- | ---- |
| 画面表示 | `signin/input/index.php` が担当する |
| 認証処理 | `signin/auth/index.php` が担当する |
| 認証ロジック | `User::auth()` が担当する |
| ログイン状態管理 | `AuthUser` が担当する |
| 保護ページ | `home/index.php` でログイン確認を行う |

## まとめ

| 項目 | 内容 |
| ---- | ---- |
| ログインフォーム | `signin/input/index.php` で表示し、入力値を再表示できる |
| 認証処理 | `signin/auth/index.php` で `POST` を受け取り、`User::auth()` を実行する |
| パスワード照合 | `password_verify()` で平文とハッシュを比較する |
| ログイン保存 | `AuthUser::set()` でログインユーザをセッションに保存する |
| ログイン確認 | `home/index.php` で `AuthUser::check()` を使う |
| ログアウト | `signout/index.php` で `AuthUser::clear()` を使う |
