## MySQL と PDO

Web アプリケーションでは、ユーザー情報や投稿データを**データベース**に保存します。
この章では **MySQL**（データベース）と **PDO**（PHP からDBに接続する仕組み）を学びます。

## MySQL とは

**MySQL** はオープンソースの**リレーショナルデータベース管理システム（RDBMS）**です。
データをテーブル（表）の形式で管理し、**SQL** という言語で操作します。

> **SQL（Structured Query Language）** = データを操作するための言語

主な SQL 操作：

| 操作 | SQL 文 | 意味 |
| ---- | -------- | -------- |
| 作成 | `CREATE` | テーブルやDBを作る |
| 読み取り | `SELECT` | データを取得する |
| 追加 | `INSERT` | データを追加する |
| 更新 | `UPDATE` | データを更新する |
| 削除 | `DELETE` | データを削除する |

## 環境設定ファイル (env.php)

DB接続に必要な情報（ホスト・ユーザー名・パスワードなど）は `env.php` に**定数**として定義します。

```php
// env.php
const DB_CONNECTION = 'mysql';
const DB_HOST       = 'localhost';
const DB_PORT       = '3306';
const DB_USERNAME   = 'root';
const DB_PASSWORD   = '';
const DB_DATABASE   = 'php_sns';
```

### なぜ env.php を分けるのか

パスワードなどの機密情報はプログラム本体とは**別ファイルに分離**して管理します。
`env.php` は `.gitignore` に追加し、Git の管理対象から除外します。

#### .gitignore
```env
env.php
```

代わりに `env.sample.php`（ダミー値入りのテンプレート）を Git に含めておくことで、
チームメンバーはそれをコピーして自分の環境に合わせた `env.php` を作ります。

> **Note**：`Composer` の `vendor/bin/` を利用すると、`.env` ファイルを直接読み込むことができ、これが一般的です。

### PDO オブジェクト
`PDO` は、PHP からデータベースに接続するためのオブジェクトです。
`new PDO()` で PDO オブジェクトを生成することで、データベースに接続できます。

```php
$pdo = new PDO(DSN, DB_USERNAME, DB_PASSWORD);
```

### 接続の流れ（db.php）

```php
require_once './env.php';

// ① DSN の組み立て
$dsn = "mysql:dbname=php_sns;host=localhost;port=3306;charset=utf8;";

// ② PDO インスタンスを生成（接続）
try {
    $pdo = new PDO($dsn, DB_USERNAME, DB_PASSWORD);

    // ③ エラーモードの設定
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
    $pdo->setAttribute(PDO::ATTR_EMULATE_PREPARES, false);

} catch (PDOException $e) {
    echo "接続失敗: " . $e->getMessage();
    exit;
}
```

### DSN（データソース名）

`DSN（Data Source Name）` とは、接続先DBを特定するための文字列です。データベースの種類は `:` の前で設定し、各値は `;` で区切ります。

```php
mysql: dbname=php_sns; host=localhost; port=3306; charset=utf8
```

| 項目 | キー | 値 |
| ---------------------------------------- | ----------------------------- | ----------------------------------------- |
| DB種類|  | `mysql` |
| DB名 | `dbname` | `php_sns` |
| ホスト | `host` |`localhost` | 
| ポート | `port` | `3306` |
| 文字コード | `charset` | `utf8` |


### setAttribute の設定
`setAttribute` で接続のオプション設定をします。

| 設定 | 値 | 意味 |
| ---------------------------------------- | ----------------------------- | ----------------------------------------- |
| `PDO::ATTR_ERRMODE` | `PDO::ERRMODE_EXCEPTION` | エラー時に例外（`PDOException`）を投げる |
| `PDO::ATTR_EMULATE_PREPARES` | `false` | プリペアドステートメントをDBに任せる（安全） |

`ERRMODE_EXCEPTION` を設定することで、SQL ミスや接続エラーを `try/catch` で確実に検知できます。


## データベースの作成（create_database.php）

`create_database.php` はボタンを押すと、次の手順を順番に実行します。

```php
// 1. データベースへの接続（DB名を指定しない）
$dsn = "mysql:host=" . DB_HOST . ";port=" . DB_PORT . ";charset=utf8mb4";
$pdo = new PDO($dsn, DB_USERNAME, DB_PASSWORD);
$pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);

// 2. データベースを作成
$pdo->exec("CREATE DATABASE IF NOT EXISTS " . DB_DATABASE);

// 3. 作成したデータベースに切り替え
$pdo->exec("USE " . DB_DATABASE);

// 4. schema.sql を読み込んでテーブルを作成
$schema_sql = file_get_contents("docs/schema.sql");
$pdo->exec($schema_sql);

// 5. insert_data.sql を読み込んで初期データを投入
$insert_sql = file_get_contents("docs/insert_data.sql");
$pdo->exec($insert_sql);
```

最初の接続では DB名（`dbname`）を DSN に含めません。
まだ存在しないDBに接続しようとするとエラーになるためです。

`CREATE DATABASE IF NOT EXISTS` を使うことで、すでにDBが存在する場合もエラーにならずに続行できます。

## スキーマ（schema.sql）

**スキーマ**とは、テーブルの設計図（構造定義）のことです。
以下は今回のアプリ（SNS）で使用するテーブル定義の抜粋です。

```sql
-- schema.sql（抜粋）

CREATE TABLE users (
    id           bigint PRIMARY KEY AUTO_INCREMENT,
    account_name varchar(255) UNIQUE NOT NULL,
    email        varchar(255) UNIQUE NOT NULL,
    display_name varchar(255) NOT NULL,
    password     varchar(255) NOT NULL,
    profile      text DEFAULT NULL,
    created_at   datetime NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at   datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

CREATE TABLE tweets (
    id         bigint PRIMARY KEY AUTO_INCREMENT,
    message    text NOT NULL,
    user_id    bigint NOT NULL,  -- users テーブルの id を参照
    created_at timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE likes (
    id        bigint PRIMARY KEY AUTO_INCREMENT,
    user_id   bigint NOT NULL,
    tweet_id  bigint NOT NULL,
    UNIQUE (user_id, tweet_id)   -- 同じ組み合わせで2度いいねできない
);
```

### よく使うカラムの型

| 型 | 意味 | 使用例 |
| ----------- | ------------------------------ | -------------------- |
| `bigint` | 大きな整数 | ID、カウント |
| `varchar(n)` | 最大 n 文字の文字列 | 名前、メール |
| `text` | 長い文章 | 本文、プロフィール |
| `datetime` | 日時（タイムゾーンなし） | created_at |
| `timestamp` | 日時（自動で現在時刻） | updated_at |

### カラムの制約

| 制約 | 意味 |
| ---------------------- | --------------------------------------- |
| `PRIMARY KEY` | 一意のID（テーブルの主キー） |
| `AUTO_INCREMENT` | 挿入するたびに自動で 1 ずつ増える |
| `NOT NULL` | NULL（空）を禁止する |
| `UNIQUE` | 重複した値を禁止する |
| `DEFAULT CURRENT_TIMESTAMP` | 挿入時に現在時刻を自動でセット |
| `ON UPDATE CURRENT_TIMESTAMP` | 更新時に現在時刻を自動でセット |

### 外部キー制約（FOREIGN KEY）

**外部キー制約**は、テーブル間の関係を定義してデータの整合性を守る仕組みです。

```sql
-- tweets.user_id は users.id を参照する
ALTER TABLE tweets ADD CONSTRAINT tweets_user_id_fk
    FOREIGN KEY (user_id) REFERENCES users(id)
    ON DELETE CASCADE   -- ユーザーを削除したらそのツイートも削除
    ON UPDATE CASCADE;  -- ユーザーIDが変わったらツイートの user_id も追随
```

`ON DELETE CASCADE` を設定すると、親テーブル（users）のレコードが削除された際に、
子テーブル（tweets / likes）の関連レコードも自動で削除されます。

## Database クラス（シングルトン）

`lib/Database.php` は、PDO 接続をクラスに**カプセル化**したモジュールです。

```php
// lib/Database.php
class Database
{
    private static $instance = null; // 唯一のインスタンスを保持
    public $pdo;

    // private: 外から new できないようにする
    private function __construct()
    {
        $dsn = "mysql:dbname=" . DB_DATABASE . ";host=" . DB_HOST . ";charset=utf8;port=" . DB_PORT;
        $this->pdo = new PDO($dsn, DB_USERNAME, DB_PASSWORD);
        $this->pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
        $this->pdo->setAttribute(PDO::ATTR_EMULATE_PREPARES, false);
    }

    // 唯一の取得口
    public static function getInstance(): PDO
    {
        if (self::$instance === null) {
            self::$instance = new Database(); // 初回のみ接続
        }
        return self::$instance->pdo; // 以降は同じ接続を返す
    }
}
```

### シングルトンパターン

**シングルトン**とは、「インスタンスを1つだけ作ることを保証する設計パターン」です。

```php
// 使い方（connect_test_for_module.php）
$pdo = Database::getInstance(); // 初回 → new Database() が走る
$pdo = Database::getInstance(); // 2回目以降 → 同じ $pdo を返す（再接続しない）
```

| ポイント | 説明 |
| ----------------------- | ------------------------------------------------ |
| `private __construct()` | 外部から `new Database()` できなくする |
| `private static $instance` | クラスが持つ唯一のインスタンスを記憶する |
| `public static getInstance()` | 外部への唯一の入口。未生成なら作り、あれば使い回す |

シングルトンを使うと、ページ内で何度 `getInstance()` を呼んでも接続が1回で済み、
パフォーマンスの無駄な消費を防げます。
