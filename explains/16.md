## MySQL と PDO

`Web` アプリケーションでは、ユーザー情報や投稿データを `データベース` に保存します。  
この章では、`MySQL` と `PDO` を使って、`PHP` から `データベース` に接続する基本を学びます。

| 用語 | 内容 |
| ---- | ---- |
| MySQL | データを表形式で管理するデータベース管理システム |
| PDO | PHP からデータベースに接続するための仕組み |

---

### MySQLとは
`MySQL` は、オープンソースの `RDBMS`（`Relational Database Management System`）です。  
データを `テーブル` の形で管理し、`SQL` を使って操作します。

> `RDBMS` は、表形式でデータを整理して管理する仕組みです。

### SQLとは
`SQL` は、`Structured Query Language` の略で、データベースを操作するための言語です。

#### 主なSQL操作
| 操作 | SQL文 | 意味 |
| ---- | ---- | ---- |
| 作成 | CREATE | データベースやテーブルを作る |
| 読み取り | SELECT | データを取得する |
| 追加 | INSERT | データを追加する |
| 更新 | UPDATE | データを更新する |
| 削除 | DELETE | データを削除する |

---

### PDOとは

`PDO` は、`PHP Data Objects` の略で、`PHP` からデータベースへ接続するための標準的な仕組みです。  
`MySQL`、`PostgreSQL`、`SQLite` など、異なる `DB` でも似た書き方で扱えます。

### 接続の全体像
<img src="/storage/teaching_material/pdo_flow.svg" class="" width="600">

### PDOオブジェクト
`PDO` は、データベース接続を表すオブジェクトです。  
`new PDO()` で生成します。

```php
$pdo = new PDO($dsn, DB_USERNAME, DB_PASSWORD);
```

### setAttributeの設定
接続後は、`setAttribute()` で動作設定を行います。

| 設定 | 値 | 意味 |
| ---- | ---- | ---- |
| PDO::ATTR_ERRMODE | PDO::ERRMODE_EXCEPTION | エラー時に例外を投げる |
| PDO::ATTR_EMULATE_PREPARES | false | プリペアドステートメントをDB側に任せる |

> `PDO::ERRMODE_EXCEPTION` を設定すると、接続エラーや `SQL` エラーを `try/catch` で検知しやすくなります。

## DSNと環境設定

### DSNの概要
`DSN` は、`Data Source Name` の略で、接続先の `DB` を特定する文字列です。  
`DB` の種類、ホスト、ポート、データベース名、文字コードなどを指定します。

### DSNの例
```php
mysql:dbname=php_sns;host=localhost;port=3306;charset=utf8mb4;
```

### DSNの構成
| 項目 | 例 | 内容 |
| ---- | ---- | ---- |
| 接続方式 | mysql | MySQL に接続する |
| dbname | php_sns | 接続するデータベース名 |
| host | localhost | 接続先ホスト |
| port | 3306 | ポート番号 |
| charset | utf8mb4 | 文字コード |

> 文字コードは `utf8` より `utf8mb4` を使う方が実務では一般的です。

---

### 環境設定ファイル
`DB` の接続情報は、`env.php` のような環境設定ファイルにまとめて管理します。  
ホスト名、ポート番号、ユーザー名、パスワード、データベース名などをコード本体から分けることで、設定変更や管理がしやすくなります。

| 理由 | 内容 |
| ---- | ---- |
| 設定を分離できる | 接続情報をプログラム本体と分けて管理できる |
| 修正しやすい | 接続先変更時に1か所だけ直せばよい |
| 環境ごとに切り替えやすい | 開発環境、本番環境で設定を分けやすい |
| 機密情報を守りやすい | パスワードなどを Git 管理から外しやすい |

> 実務では `.env` ファイルを使うことも多いですが、この教材ではまず `env.php` で仕組みを理解します。

開発環境と本番環境で `DB` の接続先が異なる場合でも、プログラム本体を変更せずに設定ファイルだけ差し替えれば対応できます。

#### env.php の設定
`DB` 接続に必要な情報は、`env.php` に定数としてまとめます。

```php
<?php
const DB_CONNECTION = 'mysql';
const DB_HOST       = 'localhost';
const DB_PORT       = '3306';
const DB_USERNAME   = 'root';
const DB_PASSWORD   = '';
const DB_DATABASE   = 'php_sns';
const DB_CHARSET    = 'utf8mb4';
```

### 設定項目
| 項目 | キー | 値の例 |
| ---- | ---- | ---- |
| DB種類 | DB_CONNECTION | mysql |
| ホスト | DB_HOST | localhost |
| ポート | DB_PORT | 3306 |
| ユーザー名 | DB_USERNAME | 環境に合わせる（XAMPP: root ）|
| パスワード | DB_PASSWORD | 環境に合わせる（XAMPP: なし） |
| DB名 | DB_DATABASE | php_sns |
| 文字コード | DB_CHARSET | utf8mb4 |

> ユーザー名、パスワードは本番環境では厳密に設定すること

#### Git管理の注意
> パスワードなどの機密情報は、プログラム本体と分離して管理します。  
> `env.php` は `.gitignore` に追加し、`Git` 管理対象から除外します。

```gitignore
env.php
```

代わりに `env.sample.php` を `Git` に含めることで、他の人はそれをコピーして自分の `env.php` を作れます。

## DB接続処理

### db.php
`db.php` では、`env.php` を読み込み、`PDO` 接続を作成します。

```php
<?php
require_once './env.php';

$db_connection = DB_CONNECTION;
$db_name       = DB_DATABASE;
$db_host       = DB_HOST;
$db_port       = DB_PORT;
$db_user       = DB_USERNAME;
$db_password   = DB_PASSWORD;
$db_charset    = DB_CHARSET;

$dsn = "{$db_connection}:dbname={$db_name};host={$db_host};port={$db_port};charset={$db_charset};";

try {
    $pdo = new PDO($dsn, $db_user, $db_password);
    $pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
    $pdo->setAttribute(PDO::ATTR_EMULATE_PREPARES, false);
} catch (PDOException $e) {
    echo "接続失敗: " . $e->getMessage();
    exit;
}
```

### 接続処理の流れ
| 手順 | 内容 |
| ---- | ---- |
| 1 | env.php を読み込む |
| 2 | 接続情報を変数に入れる |
| 3 | DSN を組み立てる |
| 4 | PDO オブジェクトを生成する |
| 5 | エラーモードなどを設定する |

### db.php を外部ファイルにする理由
`db.php` を外部ファイルに分けておくと、`PDO` の接続処理を1か所にまとめて管理できます。

| 理由 | 内容 |
| ---- | ---- |
| 再利用しやすい | 複数のファイルから同じ接続処理を使える |
| 修正しやすい | 接続設定の変更を1か所で済ませられる |
| ミスを減らせる | 接続処理の書き間違いや設定漏れを防ぎやすい |
| 見通しがよい | 各ファイルが本来の処理に集中できる |

> 接続処理を毎回各ファイルに直接書くより、`db.php` にまとめた方が保守しやすくなります。

## スキーマとは

### スキーマの概要
`スキーマ` とは、テーブルの構造定義のことです。  
どのカラムを持つか、どの型を使うか、どんな制約を付けるかを決めます。

### schema.sqlの例
```sql
CREATE TABLE users (
    id           bigint PRIMARY KEY AUTO_INCREMENT,
    account_name varchar(255) UNIQUE NOT NULL,
    email        varchar(255) UNIQUE NOT NULL,
    display_name varchar(255) NOT NULL,
    password     varchar(255) NOT NULL,
    profile      text DEFAULT NULL,
    created_at   datetime NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at   datetime NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### よく使うカラム型
| 型 | 意味 | 使用例 |
| ---- | ---- | ---- |
| bigint | 大きな整数 | ID |
| varchar(n) | 最大 n 文字の文字列 | 名前、メール |
| text | 長い文章 | 本文、プロフィール |
| datetime | 日時 | created_at |
| timestamp | 日時 | updated_at |

### カラムの制約
| 制約 | 意味 |
| ---- | ---- |
| PRIMARY KEY | 一意のID |
| AUTO_INCREMENT | 自動で連番を振る |
| NOT NULL | NULLを禁止する |
| UNIQUE | 重複を禁止する |
| DEFAULT CURRENT_TIMESTAMP | 作成時に現在時刻を入れる |
| ON UPDATE CURRENT_TIMESTAMP | 更新時に現在時刻を更新する |

### 外部キー制約
`外部キー制約` は、テーブル同士の関係を定義し、データの整合性を守る仕組みです。

```sql
ALTER TABLE tweets ADD CONSTRAINT tweets_user_id_fk
    FOREIGN KEY (user_id) REFERENCES users(id)
    ON DELETE CASCADE
    ON UPDATE CASCADE;
```

### 外部キーの意味
| 設定 | 意味 |
| ---- | ---- |
| FOREIGN KEY | 他のテーブルを参照する |
| ON DELETE CASCADE | 親が削除されたら子も削除する |
| ON UPDATE CASCADE | 親のID変更に合わせて子も更新する |

> `CASCADE` を使うと関連データも自動で更新・削除されます。

## データベースの作成

### create_database.phpの役割
`create_database.php` は、データベース作成、テーブル作成、初期データ投入を一括で行うプログラムです。

### 処理の流れ
| 手順 | 内容 |
| ---- | ---- |
| 1 | MySQL サーバへ接続する |
| 2 | データベースを作成する |
| 3 | 作成したデータベースを選択する |
| 4 | schema.sql を実行する |
| 5 | insert_data.sql を実行する |

### なぜ最初はdbnameを入れないのか
最初の接続では、まだ `php_sns` が存在しない可能性があります。  
そのため、最初の `DSN` には `dbname` を含めません。

```php
$dsn = "mysql:host=" . DB_HOST . ";port=" . DB_PORT . ";charset=utf8mb4";
```

その後で、`CREATE DATABASE IF NOT EXISTS` を実行します。

> 存在しない `DB` に最初から接続しようとすると、エラーになります。

`connect_test.php` では、`db.php` を読み込んで接続確認を行います。  
接続できた場合は `PDO` オブジェクトを取得し、画面に接続成功を表示します。

### テストのポイント

## Databaseクラス（シングルトン）

### シングルトンパターンとは
`シングルトン` とは、**インスタンスを1つだけ作る設計パターン**です。  
データベース接続は何度も作る必要がないため、この考え方と相性が良いです。

### ポイント
| ポイント | 説明 |
| ---- | ---- |
| private __construct() | 外部から new できなくする |
| private static $instance | 唯一のインスタンスを保持する |
| public static getInstance() | 取得用の唯一の入口にする |

### Database.php
```php
<?php
class Database
{
    private static $instance = null;
    public $pdo;

    private function __construct()
    {
        $dsn = "mysql:dbname=" . DB_DATABASE . ";host=" . DB_HOST . ";charset=" . DB_CHARSET . ";port=" . DB_PORT;
        $this->pdo = new PDO($dsn, DB_USERNAME, DB_PASSWORD);
        $this->pdo->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
        $this->pdo->setAttribute(PDO::ATTR_EMULATE_PREPARES, false);
    }

    public static function getInstance(): PDO
    {
        if (self::$instance === null) {
            self::$instance = new Database();
        }
        return self::$instance->pdo;
    }
}
```

### 使い方
```php
$pdo = Database::getInstance();
```

> 何度 `getInstance()` を呼んでも、同じ接続を使い回す設計です。

## まとめ

### ファイル構成
```treeview
./
├── env.php                        ← DB接続情報（Gitに含めない）
├── env.sample.php                 ← env.php のテンプレート
├── db.php                         ← PDO接続処理
├── connect_test.php               ← db.php を使った接続確認
├── connect_test_for_module.php    ← Database クラスを使った接続確認
├── create_database.php            ← DB作成・スキーマ適用・初期データ投入
├── lib/
│   └── Database.php               ← シングルトンの DB クラス
└── docs/
    ├── schema.sql                 ← テーブル定義 SQL
    └── insert_data.sql            ← 初期データ投入 SQL
```

### 各ファイルの役割
| ファイル | 役割 |
| ---- | ---- |
| env.php | ホスト、ユーザー名、パスワードなどの接続定数 |
| env.sample.php | 接続情報テンプレート |
| db.php | PDO オブジェクトを生成して接続する |
| connect_test.php | db.php を使った接続確認ページ |
| connect_test_for_module.php | Database クラスを使った接続確認ページ |
| create_database.php | DB・テーブル・初期データをまとめて作成する |
| docs/schema.sql | CREATE TABLE などのテーブル定義 |
| docs/insert_data.sql | 初期データ投入用 SQL |

### 接続まわりの重要点
| 項目 | 内容 |
| ---- | ---- |
| DSN | 接続先情報を文字列で表す |
| PDO | PHP から DB に接続する |
| env.php | 接続情報を分離して管理する |
| utf8mb4 | 文字化け防止のために使う文字コード |
| try/catch | 接続エラーを安全に扱う |
| Databaseクラス | 接続処理をまとめて再利用する |

### 用語
| 概念 | キーワード | 意味 |
| ---- | ---- | ---- |
| データベース | MySQL | データを表形式で管理する仕組み |
| 接続 | PDO / DSN | PHP から DB に接続する仕組み |
| 環境設定 | env.php / const | 接続情報を分離して管理する |
| テーブル定義 | CREATE TABLE / スキーマ | テーブル構造を定義する |
| 関係 | FOREIGN KEY / CASCADE | テーブル同士の参照関係を定義する |
| クラス化 | Database / シングルトン | 接続処理をまとめて使い回す |
