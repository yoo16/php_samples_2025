## 7：Laravelとの比較・自分の機能を実装する

### 講義（20分）

**このアプリとLaravelの比較**

| 機能 | このアプリ | Laravel |
|---|---|---|
| ルーティング | ディレクトリ構造 / router.php | `routes/web.php` |
| ブートストラップ | `app.php` を手動 require | 自動（フレームワークが処理） |
| DBアクセス | PDO を直接記述 | Eloquent ORM |
| 認証 | `AuthUser::checkLogin()` | `auth` ミドルウェア |
| View | `Request::render()` | `view()` ヘルパー |
| MVCの場所 | `app/controllers/` `app/models/` `app/views/` | `app/Http/Controllers/` `app/Models/` `resources/views/` |

**MVCの概念は同じ**

Laravelを学ぶときに「このアプリのどれに対応するか」を考えると理解が速い。

```
このアプリ                Laravel
app.php          →   bootstrap/app.php（自動）
HomeController   →   HomeController（ほぼ同じ）
Tweet モデル     →   Tweet モデル（Eloquentに変わる）
.view.php        →   .blade.php（テンプレートエンジン）
router.php       →   routes/web.php
```

**フレームワークを使う理由**

- ルーティング・認証・バリデーションを毎回書かなくていい
- セキュリティのベストプラクティスが組み込まれている
- チーム開発で構造が統一される

「このアプリで手書きした部分をフレームワークが代わりにやってくれる」

### 演習（30分）

## 8：発表・コードレビュー

### 発表（40分）

### コードレビュー（10分）

教師が2〜3名のコードをピックアップして講評する。

**レビューの観点**
- プリペアドステートメントが使われているか（SQLインジェクション対策）
- 出力に `h()` / `sanitize()` が使われているか（XSS対策）
- 認証チェックが実装されているか
- 既存コードのスタイルと統一されているか
