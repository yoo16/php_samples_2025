## 2：MVCの役割分担を理解する

### MVCの概念図

* **Controller (C)**: ウェイター。注文（リクエスト）を受け、キッチンに指示を出す。
* **Model (M)**: シェフ。材料（データ）を冷蔵庫（DB）から取り出し、調理する。
* **View (V)**: 盛り付け。出来上がった料理を、客（ブラウザ）が見やすい皿に盛る。

### 🔍 処理の流れ（home/index.php の場合）

ユーザーが「ホーム画面」にアクセスしたとき、裏側では **Action → Controller → Model → View** という4つの役割がバトンをつないで画面を表示しています。

#### 1. 入口（Action）: `home/index.php`
全ての始まりはこのファイルです。ユーザーの「ホーム画面を見たい」というリクエストを受け取り、司令塔（Controller）を呼び出します。

```php
// app.php で必要な設定やクラスを全て読み込む
require_once '../app.php';

// 司令塔（HomeController）を生成して「index」という仕事を頼む
$controller = new HomeController();
$controller->index();
```

#### 2. 司令塔（Controller）: `app/controllers/HomeController.php`
コントローラーは「何を表示するか」の判断をします。ログインしているか確認し、必要なデータを揃えて「見た目（View）」に渡します。

```php
public function index()
{
    // ログインユーザーの情報を取得（AuthUserモデルを利用）
    $auth_user = AuthUser::get();
    
    // 画面の「型」を決めて、ユーザー情報を渡す
    // ※実際のツイートデータは、画面が表示された後にJavaScript経由で取得されます
    Request::render('home/index', ['auth_user' => $auth_user]);
}
```

#### 3. データ担当（Model）: `app/models/Tweet.php`
モデルは「DB（データベース）とのやり取り」の専門家です。ツイートの取得や保存を行います。

```php
// 例：ツイートを10件取得する命令
public function get($limit = 10, $offset = 0)
{
    // SQLを発行してデータベースからツイートを取り出す
    // (HomeController.php や API から呼び出される)
}
```

#### 4. 見た目担当（View）: `app/views/home/index.view.php`
ビューは、コントローラーから受け取ったデータを使って、ユーザーが見るHTML（画面）を作成します。

```php
<!-- ヘッダーやナビゲーションを表示 -->
<?php include COMPONENT_DIR . 'nav.php' ?>

<!-- ツイート投稿フォームを表示 -->
<?php include COMPONENT_DIR . 'tweet_form.php' ?>

<!-- ここに JavaScript が Model(Tweet) を通じて取得したツイートが表示される -->
<div id="tweet-list" data-auth-user-id="<?= $auth_user['id'] ?>">
    <!-- ローディング表示など -->
</div>
```

---

### 「役割」まとめ

| 役割 | ファイル | 例え | 仕事の内容 |
| --- | --- | --- | --- |
| **Action** | `home/index.php` | **玄関** | お客さん（ユーザー）を迎え入れる。 |
| **Controller** | `HomeController.php` | **ウェイター** | 注文を受け、シェフ(Model)に指示し、盛り付け(View)を頼む。 |
| **Model** | `Tweet.php` | **シェフ** | 冷蔵庫（DB）から材料（データ）を取り出して調理する。 |
| **View** | `index.view.php` | **盛り付け** | 出来上がった料理を、きれいに皿に盛って客に出す。 |

## 🛠 アプリを支える共通パーツ

MVCの3つの役割以外にも、アプリ全体で使い回す便利なパーツや設定ファイルがあります。

### ⚙️ `env.php`：環境設定（秘密のメモ）
データベースの接続情報（ホスト名、ユーザー名、パスワード）など、**アプリを動かす環境に合わせて書き換える必要がある情報**をまとめています。
#### 役割
データベースへ接続情報やAPIのキーなど、**アプリの個別設定を管理**する。
開発環境（自分のPC）と本番環境（サーバー）で内容が変わるため、`.gitignore` に追加して共有されないようにするのが一般的です。

### 📦 `app.php`：アプリの起動スイッチ（全体設定）
全てのページ（Action）で最初に読み込まれる、**アプリの「心臓部」**です。
#### 役割
`env.php` を読み込み、セッション（ログイン状態の維持）を開始し、必要なクラスファイルを一気に読み込みます。
どこに何のファイルがあるか（ディレクトリパス）などの共通ルールもここで定義されています。

### 📚 `lib/` ディレクトリ：共通ライブラリ（便利な道具箱）
どのページでも使うような「便利な機能」を詰め込んだ道具箱です。

| ファイル名 | 役割 | 内容の例 |
| --- | --- | --- |
| `Database.php` | データベース接続 | DBとの「つなぎ役」。どのモデルからも共通で使われます。 |
| `Sanitize.php` | 安全対策（サニタイズ） | ユーザーが入力した危険な文字（タグなど）を無害化します。 |
| `File.php` | ファイル操作 | 画像のアップロード処理や、保存先の管理を行います。 |
| `Request.php` | 通信・表示の補助 | 画面の表示（レンダリング）や、POST通信の判定などを行います。 |
