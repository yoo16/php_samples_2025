## オブジェクト指向とは

### オブジェクトが基本

ゲームで登場する「カード」を1つの実体として考えます。
このカードには「名前」「攻撃力」「HP」などの**特徴**があり、
「攻撃する」「レベルアップする」などの**動作**があります。

このように、

> **特徴（データ）＋動作（処理）をまとめたものがオブジェクト**

です。

今回の例では、カードそのものがオブジェクトになります。

---

### オブジェクトで役割分担する

カードバトルゲームを考えてみましょう。
まずプログラミングをする前に、カードバトルゲームの役割を考えてみましょう。

| オブジェクト | 役割 |
| ------ | ------------ |
| カード | 戦闘・成長・スキルを持つ |
| プレイヤー | カードを操作する |
| バトル | ダメージ計算や勝敗判定 |
| ゲーム | ゲームの流れを管理する |
| UI | ゲーム画面を表示する |

オブジェクト指向では、これらの役割をオブジェクトとして表現します。

---

## クラスとは

クラスとは **オブジェクトの設計図**で、実際にオブジェクトを作るためのひな形です。
今回のプログラムでは、**カードの基本設計**を担当する `BaseCard` がクラスです。

### クラス定義
クラスを定義するには、`class` キーワードを用いてクラス名 `BaseCard` を定義します。

```php
abstract class BaseCard implements CardInterface
{
    // ここにプロパティやメソッドを書く
}
```

### プロパティ（メンバ変数）

クラスには、そのクラスの特徴となるデータを定義します。
これらは **プロパティ（メンバ変数）** と呼ばれます。

| 項目 | 意味 |
| ----------------- | ----------- |
| $level | レベル |
| $name | 名前 |
| $attack | 攻撃力 |
| $defense | 防御力 |
| $hp / $maxHp | 現在HP / 最大HP |
| $mp / $maxMp | 現在MP / 最大MP |
| $exp | 経験値 |
| $element | 属性 |
| $image | 画像パス |
| $specialSkill | 必殺技名 |
| $specialSkillPower | 必殺技威力 |

```php
abstract class BaseCard implements CardInterface
{
    // プロパティ
    public int $level = 1;
    public string $name = '';
    public int $attack = 0;
    public int $defense = 0;
    public int $hp = 0;
    public int $maxHp = 0;
    // ... (省略)
}
```

---

## インスタンス

クラス（設計図）から作られた実体を **インスタンス** といいます。
`new` キーワードを使って、設計図から具体的なオブジェクトを生成します。

`instance.php` では、次のように `BaseCard` クラスからインスタンスを作成しています。

```php
// BaseCard クラスからインスタンスを生成
$card = new BaseCard(
    '水の精霊',      // 名前
    18,             // 攻撃力
    25,             // 防御力
    100,            // HP
    3,              // MP
    '水',           // 属性
    'AquaCard.png', // 画像
    'ハイドロポンプ', // 必殺技
    35              // 必殺技威力
);
```

この `$card` が、メモリ上に実体として作られた「カードオブジェクト」です。
※本来 `BaseCard` は `abstract`（抽象クラス）なので直接 `new` できませんが、まずは基本として「クラスからインスタンスを作る」流れを理解しましょう。

---

## クラスメソッド

クラスには、そのクラスの動作（機能）を定義します。
これらは **クラスメソッド** と呼ばれます。


| メソッド | 役割 |
| ---------------- | -------- |
| `showStatus()` | ステータス表示 |
| `attack($target)` | 通常攻撃（相手にダメージを与える） |
| `specialSkill($target)` | 必殺技 |
| `gainExp($exp)` | 経験値取得 |
| `isLevelUp()` | レベルアップ判定 |
| `levelUp()` | レベルアップ処理 |

### `$this` の使い方
メソッドの中で自分自身のプロパティにアクセスするときは、`$this->プロパティ名` と書きます。

```php
public function gainExp(int $exp): void
{
    // 自分の exp プロパティに引数の $exp を加算する
    $this->exp += $exp;
}
```

## コンストラクタとは？

```php
public function __construct(...)
```

これは、インスタンス生成時（new した時）に**自動実行される特別なメソッド**です。
主にプロパティの初期化に使われます。

### 親クラス（BaseCard）のコンストラクタ
共通の初期化処理を書いておきます。

```php
public function __construct(string $name, int $attack, ...) {
    $this->name = $name;
    $this->attack = $attack;
    // ...
}
```

---

## 継承 (extends)

**継承**とは、あるクラスの機能を引き継いで、新しいクラスを作ることです。

* **親クラス (BaseCard)**: 全てのカードに共通する機能（攻撃、レベルアップなど）を持つ。
* **子クラス (AquaCard, ForestCard など)**: 親の機能を引き継ぎつつ、個別の名前やステータスを持つ。

```php
class AquaCard extends BaseCard // BaseCard を継承
```

これによって、`AquaCard` は自分で `attack()` メソッドを書かなくても、`BaseCard` が持っている `attack()` を使うことができます。

### 子クラス（AquaCard）のコンストラクタ
`parent::__construct()` を使って、親クラスのコンストラクタを呼び出します。これにより、共通の初期化処理を再利用できます。

```php
class AquaCard extends BaseCard {
    public function __construct() {
        // 親クラスのコンストラクタに具体的な値を渡して呼び出す
        parent::__construct('水の精霊', 18, 25, 100, 3, ...);
    }
}
```

---

## インターフェース (implements)

**インターフェース**は、クラスが実装すべきメソッドを定義した「規約」です。

```php
interface CardInterface {
    public function attack(BaseCard $target): int;
    // ...
}
```

`implements CardInterface` と書くことで、「このクラスは必ず `attack` メソッドを持っています」ということを保証します。

---

## なぜ abstract なのか？

`BaseCard` クラスには `abstract`（抽象）キーワードが付いています。

```php
abstract class BaseCard implements CardInterface
```

これは、「このクラスは設計図のベース（親クラス）であり、**直接インスタンス化（実体化）してはいけない**」という意味です。

```php
$card = new BaseCard(...); // ❌ エラー：抽象クラスは実体化できない
$card = new AquaCard();    // ✅ 継承した具体的なクラスなら実体化できる
```

「ただのカード」という実体は存在せず、必ず「水のカード」や「森のカード」として実体化させる、というルールを表現しています。

---

## インスタンス

クラスから作られた実体を **インスタンス** といいます。

```php
// AquaCard クラスのインスタンスを生成
$aquaCard = new AquaCard();

// メソッドを呼び出す
$aquaCard->gainExp(50);
```

この `$aquaCard` がメモリ上に作られた実体（インスタンス）です。

---

## まとめ

* **クラス** = 設計図（class）
* **インスタンス** = 設計図から作られた実体（new）
* **プロパティ** = オブジェクトが持つデータ（変数）
* **メソッド** = オブジェクトができること（関数）
* **継承 (extends)** = 他のクラスの機能を引き継ぐ
* **抽象クラス (abstract)** = 継承専用のベースとなるクラス
* **インターフェース (implements)** = 実装すべき機能のルール（規約）
